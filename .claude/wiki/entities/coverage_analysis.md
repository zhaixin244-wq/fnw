# coverage_analysis

> 验证覆盖率分析，代码覆盖率、功能覆盖率、断言覆盖率

## 基本信息

| 字段 | 值 |
|------|----|
| 类型 | 概念 |
| 来源 | .claude/knowledge/verification/coverage_analysis.md |

## 核心特性
- 代码覆盖率：行覆盖、分支覆盖、条件覆盖、FSM 覆盖
- 功能覆盖率：用户定义的功能点覆盖
- 断言覆盖率：SVA 断言触发程度
- 覆盖率驱动验证收敛

## 关键参数

| 覆盖率类型 | 目标值 | 说明 |
|-----------|--------|------|
| 行覆盖率 | > 95% | 代码行执行比例 |
| 分支覆盖率 | > 90% | 分支执行比例 |
| 条件覆盖率 | > 85% | 条件组合覆盖 |
| FSM 覆盖率 | > 95% | 状态和转移覆盖 |
| 功能覆盖率 | > 95% | 用户定义覆盖点 |
| 断言覆盖率 | > 90% | 断言触发比例 |

## 典型应用场景
- 验证质量评估
- 验证收敛分析
- 测试用例有效性评估
- 验证计划追踪

## 与其他实体的关系
- **simulation_basics**：仿真产生覆盖率数据
- **uvm_basics**：UVM 功能覆盖率模型
- **formal_basics**：形式验证覆盖率
- **verification_overview**：覆盖率驱动验证方法

## 设计注意事项
- 代码覆盖率高不等于功能验证充分
- 功能覆盖率需要与验证计划对齐
- 覆盖率洞（Coverage Hole）需要分析原因
- 覆盖率收敛是验证完成的重要指标
