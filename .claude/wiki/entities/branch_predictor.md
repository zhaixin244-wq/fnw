# branch_predictor

> 处理器分支预测器架构设计，直接影响流水线效率和 IPC

## 基本信息

| 字段 | 值 |
|------|----|
| 类型 | 概念 |
| 来源 | .claude/knowledge/cpu/branch_predictor.md |

## 核心特性
- 程序中 15-25% 指令是分支，每 1% 准确率提升带来 ~1% IPC 提升
- 静态预测：总是不跳转/跳转、后向跳转/前向不跳转（准确率 50-60%）
- 动态预测：1-bit/2-bit 饱和计数器、GShare、TAGE、感知机预测器
- 预测失败惩罚：2-5+ 周期（取决于流水线深度）

## 关键参数

| 预测器类型 | 准确率 | 硬件开销 | 说明 |
|------------|--------|----------|------|
| 静态 | 50-60% | 无 | 最简单 |
| 2-bit 饱和 | 80-85% | 极小 | BHT |
| GShare | 90-93% | 小 | 全局历史 |
| TAGE | 95-97% | 中 | 多历史长度 |
| 感知机 | 96-98% | 大 | 神经网络方法 |

## 典型应用场景
- 高性能 CPU 微架构设计
- 超标量处理器
- 深流水线处理器

## 与其他实体的关系
- **pipeline**：控制冒险的解决方案
- **lfsr**：伪随机数用于预测器索引

## 设计注意事项
- BTB（Branch Target Buffer）缓存分支目标地址
- RAS（Return Address Stack）处理函数返回
- 间接分支需要单独的预测器
