## 核心职责

1. **子模块收集**：收集所有子模块的端口列表和接口定义
2. **接口对齐**：检查子模块间接口信号的一致性（命名、位宽、方向）
3. **顶层模块编写**：编写顶层模块，仅做实例化和连线
4. **系统级 Lint**：对完整设计运行 lint 检查
5. **集成验证**：检查模块间连接的正确性
6. **连接图生成**：生成模块连接拓扑图

## 工作流程

### Step 1：子模块端口收集

读取所有子模块微架构文档的端口列表：

| 子模块 | 端口数 | 输入 | 输出 | 接口类型 |
|--------|--------|------|------|----------|
| {sub1} | {N} | {N} | {N} | {协议} |
| {sub2} | {N} | {N} | {N} | {协议} |

### Step 2：接口一致性检查

**检查项**：

| 检查项 | 说明 | 判定标准 |
|--------|------|----------|
| 信号命名 | 上下游同名信号是否一致 | 完全匹配 |
| 位宽 | 数据/地址位宽是否匹配 | 完全匹配 |
| 方向 | 源模块输出 = 目标模块输入 | 方向正确 |
| 时钟域 | 跨模块信号时钟域是否一致 | 一致或有 CDC |
| 复位策略 | 复位极性、同步/异步是否一致 | 统一 |
| 协议 | 握手协议（valid/ready）是否匹配 | 协议一致 |

**冲突处理**：
- 命名冲突 → 以 FS 定义为准
- 位宽冲突 → 以数据通路最宽为准，窄端做适配
- 协议冲突 → 以微架构文档为准，需增加适配逻辑

### Step 3：顶层模块编写

**铁律：顶层模块仅做子模块实例化和信号连接，禁止包含任何逻辑。**

```verilog
module {module}_top #(
    parameter DATA_WIDTH = 32
)(
    // 外部接口
    input  wire clk,
    input  wire rst_n,
    // ... 其他端口
);

    // 子模块实例化（名称关联）
    {sub1} u_{sub1} (
        .clk        (clk),
        .rst_n      (rst_n),
        .{signal}   ({signal}),
        // ...
    );

    {sub2} u_{sub2} (
        .clk        (clk),
        .rst_n      (rst_n),
        .{signal}   ({signal}),
        // ...
    );

endmodule
```

### Step 4：连接图生成

使用 `chip-png-d2-gen` 生成模块连接图（`wd_{module}_top_connect.d2`）：
- 子模块作为节点
- 外部接口作为边界节点
- 连线标注接口名称
- 配色：子模块/蓝色系，外部输入/绿色系，外部输出/红色系

### Step 5：系统级 Lint

```bash
verilator --lint-only -Wall {all_submodules}.v {module}_top.v
```

重点关注：
- 未连接端口
- 位宽不匹配
- 多驱动信号
- 时钟域混用

### Step 6：集成验证

检查项：
- [ ] 所有子模块已实例化
- [ ] 所有外部端口已连接
- [ ] 无悬空信号
- [ ] 无多余连线
- [ ] 时钟/复位统一
- [ ] 子模块间信号方向正确
- [ ] 参数传递正确
