# DDR IP

> **用途**：DDR 控制器 IP 选型参考，供 SoC 架构设计时检索
> **典型应用**：所有需要内存的 SoC

---

## 概述

DDR（Double Data Rate）是主流的内存标准，DDR 控制器是 SoC 的关键组件。

### DDR 版本对比

| 版本 | 速率 | 带宽 | 电压 | 典型应用 |
|------|------|------|------|----------|
| DDR3 | 800-2133 MT/s | 6.4-17 GB/s | 1.5V | 旧系统 |
| DDR4 | 1600-3200 MT/s | 12.8-25.6 GB/s | 1.2V | 服务器、PC |
| DDR5 | 3200-6400 MT/s | 25.6-51.2 GB/s | 1.1V | 新服务器 |
| LPDDR4 | 3200 MT/s | 25.6 GB/s | 1.1V | 移动设备 |
| LPDDR5 | 6400 MT/s | 51.2 GB/s | 1.05V | 移动设备 |
| LPDDR5X | 8533 MT/s | 68.2 GB/s | 1.05V | 旗舰手机 |

### 带宽对比

| 版本 | 通道 | 总带宽 |
|------|------|--------|
| DDR4-3200 | 1 | 25.6 GB/s |
| DDR4-3200 | 2 | 51.2 GB/s |
| DDR5-6400 | 1 | 51.2 GB/s |
| DDR5-6400 | 2 | 102.4 GB/s |
| LPDDR5X-8533 | 1 | 68.2 GB/s |
| LPDDR5X-8533 | 4 | 272.8 GB/s |

## DDR 配置选项

### 基础配置

| 配置 | 选项 | 权衡 |
|------|------|------|
| 版本 | DDR4/5、LPDDR4/5/5X | 性能 vs 功耗 |
| 通道数 | 1-4 | 带宽 vs 面积 |
| 数据宽度 | 8/16/32/64-bit | 带宽 vs 引脚数 |
| 频率 | 1600-6400 MT/s | 性能 vs 功耗 |

### 高级配置

| 配置 | 选项 | 说明 |
|------|------|------|
| ECC | 支持/不支持 | 错误检测和纠正 |
| RAS | 支持/不支持 | 可靠性、可用性、可维护性 |
| 多通道 | 独立/交织 | 带宽 vs 复杂度 |
| 优先级 | 固定/可配置 | QoS 支持 |
| 加密 | 支持/不支持 | 数据安全 |

## DDR 控制器架构

### 基本架构

```
┌─────────────────────────────────────────────┐
│  DDR 控制器                                 │
│  ┌──────────────────────────────────────┐   │
│  │  AXI 接口                            │   │
│  │  (接收 SoC 请求)                     │   │
│  └──────────────────────────────────────┘   │
│  ┌──────────────────────────────────────┐   │
│  │  请求队列                            │   │
│  │  (缓存请求)                          │   │
│  └──────────────────────────────────────┘   │
│  ┌──────────────────────────────────────┐   │
│  │  调度器                              │   │
│  │  (优化访问顺序)                      │   │
│  └──────────────────────────────────────┘   │
│  ┌──────────────────────────────────────┐   │
│  │  DDR PHY                             │   │
│  │  (物理层接口)                        │   │
│  └──────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

### 调度算法

| 算法 | 说明 | 优势 | 劣势 |
|------|------|------|------|
| FIFO | 先进先出 | 简单 | 性能低 |
| FR-FCFS | 先行请求先服务 | 性能高 | 复杂 |
| 优先级 | 基于优先级 | QoS 支持 | 复杂 |
| 自适应 | 根据负载调整 | 最优 | 最复杂 |

## DDR PHY

### PHY 功能

- **时钟生成**：生成 DDR 时钟
- **数据选通**：生成 DQS 信号
- **数据缓冲**：缓存读写数据
- **校准**：校准时序和电压

### PHY 配置

| 配置 | 选项 | 说明 |
|------|------|------|
| 数据宽度 | 8/16/32/64-bit | 与 DDR 颗粒匹配 |
| 频率 | 最大支持频率 | 决定带宽 |
| 校准 | 支持/不支持 | 时序校准 |
| ODT | 支持/不支持 | 终端电阻 |

## 选型建议

### 嵌入式

- **推荐**：ARM DDR4
- **配置**：1 通道，32-bit，DDR4-2400
- **理由**：面积小，功耗低

### 移动设备

- **推荐**：Synopsys/Cadence LPDDR5
- **配置**：4 通道，16-bit，LPDDR5-6400
- **理由**：高带宽，低功耗

### 服务器

- **推荐**：Synopsys/Cadence DDR5
- **配置**：2-4 通道，64-bit，DDR5-6400，ECC
- **理由**：高带宽，高可靠性

### AI 加速器

- **推荐**：Synopsys DDR5/HBM
- **配置**：4 通道，DDR5-6400，或 HBM2E
- **理由**：极高带宽

## 集成注意事项

### 接口匹配

- **AXI 接口**：确保与 SoC 互联匹配
- **时钟域**：DDR 时钟与系统时钟同步
- **中断**：ECC 错误中断

### 验证策略

- **协议验证**：DDR 协议一致性
- **性能验证**：带宽和延迟测试
- **兼容性**：与 DDR 颗粒兼容

### 物理集成

- **PHY 布局**：高速 PHY 布局约束
- **电源**：独立电源域
- **DFT**：扫描链、BIST 集成
