# 多核架构设计

> **用途**：多核处理器架构设计参考，供多核 SoC/CPU 子系统设计时检索
> **典型应用**：多核 SoC、服务器 CPU、AI 加速器

---

## 概述

多核处理器通过在同一芯片上集成多个处理核心来提升并行处理能力。

### 多核优势

- **并行处理**：多线程/多任务并行执行
- **能效比**：多低频核 vs 少高频核
- **可扩展**：根据需求增减核心数
- **共享资源**：共享缓存、内存控制器

---

## 多核拓扑

### 1. 总线（Bus）

```
┌─────────┐  ┌─────────┐  ┌─────────┐
│  Core 0 │  │  Core 1 │  │  Core 2 │
└────┬────┘  └────┬────┘  └────┬────┘
     │            │            │
     └────────────┼────────────┘
                  │
           ┌──────▼──────┐
           │   System Bus│
           └──────┬──────┘
                  │
           ┌──────▼──────┐
           │   Memory    │
           └─────────────┘
```

- **优点**：简单，广播天然支持
- **缺点**：带宽有限，不可扩展

### 2. 交叉开关（Crossbar）

```
┌─────────┐  ┌─────────┐  ┌─────────┐
│  Core 0 │  │  Core 1 │  │  Core 2 │
└────┬────┘  └────┬────┘  └────┬────┘
     │            │            │
     └────────────┼────────────┘
                  │
           ┌──────▼──────┐
           │  Crossbar   │
           └──────┬──────┘
                  │
    ┌─────────────┼─────────────┐
    │             │             │
┌───▼───┐   ┌────▼────┐   ┌────▼────┐
│Mem Ctl│   │ L3 Cache│   │  I/O    │
└───────┘   └─────────┘   └─────────┘
```

- **优点**：全连接，高带宽
- **缺点**：面积大，功耗高

### 3. 环形（Ring）

```
┌─────────┐    ┌─────────┐    ┌─────────┐
│  Core 0 │───→│  Core 1 │───→│  Core 2 │
└─────────┘    └─────────┘    └─────────┘
     ↑                              │
     └──────────────────────────────┘
```

- **优点**：可扩展，布线简单
- **缺点**：延迟随跳数增加

### 4. 网格（Mesh）

```
┌─────────┐    ┌─────────┐    ┌─────────┐
│  Core 0 │───→│  Core 1 │───→│  Core 2 │
└────┬────┘    └────┬────┘    └────┬────┘
     │              │              │
     ↓              ↓              ↓
┌─────────┐    ┌─────────┐    ┌─────────┐
│  Core 3 │───→│  Core 4 │───→│  Core 5 │
└─────────┘    └─────────┘    └─────────┘
```

- **优点**：高可扩展性，局部通信高效
- **缺点**：远距离通信延迟高

### 拓扑对比

| 拓扑 | 带宽 | 延迟 | 面积 | 可扩展性 |
|------|------|------|------|----------|
| 总线 | 低 | 低 | 小 | 差 |
| 交叉开关 | 高 | 低 | 大 | 中 |
| 环形 | 中 | 中 | 中 | 好 |
| 网格 | 高 | 中 | 中 | 最好 |

## 缓存一致性

### 监听协议（Snooping）

#### MESI 协议

| 状态 | Modified | Exclusive | Shared | Invalid |
|------|----------|-----------|--------|---------|
| 本地读 | 命中 | 命中 | 命中 | 未命中 |
| 本地写 | 命中 | 命中 | 需 Invalidate | 需取数据 |
| 远程读 | 写回+共享 | 转为共享 | 共享 | 无动作 |
| 远程写 | 写回+无效 | 无效 | 无效 | 无动作 |

#### 状态转换

```
         本地读           本地写
I ──────────────→ S ──────────────→ M
│                 │                 │
│  远程读          │  远程读          │  远程读
↓                 ↓                 ↓
I               S ──────────────→ S
                 (写回+共享)
```

### 目录协议（Directory）

```
┌─────────────────────────────────────────┐
│  Directory Entry                        │
│  ┌─────────┬─────────┬───────────────┐  │
│  │  State  │  Sharers│  Owner        │  │
│  ├─────────┼─────────┼───────────────┤  │
│  │  Uncached│ 000    │  -            │  │
│  │  Shared │ 011    │  -            │  │
│  │  Owned  │ 011    │  Core 0       │  │
│  │  Modified│ 001   │  Core 0       │  │
│  └─────────┴─────────┴───────────────┘  │
└─────────────────────────────────────────┘
```

- **Uncached**：无核缓存该块
- **Shared**：多个核缓存，共享状态
- **Owned**：一个核拥有，可写，其他核可能有共享副本
- **Modified**：一个核独占且脏

### 监听 vs 目录

| 特性 | 监听协议 | 目录协议 |
|------|----------|----------|
| 带宽 | 高（广播） | 低（点对点） |
| 延迟 | 低 | 中 |
| 可扩展性 | 差 | 好 |
| 面积 | 小 | 大（需目录存储） |
| 适用 | 小核数（2-8） | 大核数（8+） |

## 片上网络（NoC）

### NoC 组件

| 组件 | 功能 |
|------|------|
| 路由器（Router） | 数据包路由 |
| 链路（Link） | 路由器间连接 |
| 网络接口（NI） | 核心/缓存接入 NoC |

### 路由算法

| 算法 | 说明 | 特点 |
|------|------|------|
| XY 路由 | 先 X 后 Y | 简单，无死锁 |
| 自适应路由 | 根据拥塞选择路径 | 灵活，需防死锁 |
| 确定性路由 | 固定路径 | 简单，可预测 |

### 流控

| 类型 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| 存储转发 | 完整包存储后转发 | 简单 | 延迟大 |
| 虫洞（Wormhole） | 分片流水传输 | 延迟小 | 需缓冲管理 |
| 虚通道（VC） | 多虚通道复用 | 防死锁 | 面积大 |

## 核间通信

### 共享内存

```
┌─────────┐  ┌─────────┐
│  Core 0 │  │  Core 1 │
└────┬────┘  └────┬────┘
     │            │
     └──────┬─────┘
            │
     ┌──────▼──────┐
     │ Shared Cache │
     │   (L3)      │
     └──────┬──────┘
            │
     ┌──────▼──────┐
     │   Memory    │
     └─────────────┘
```

- 通过共享内存交换数据
- 需要一致性协议
- 编程模型简单

### 消息传递

```
┌─────────┐  ┌─────────┐
│  Core 0 │  │  Core 1 │
└────┬────┘  └────┬────┘
     │            │
     │  Mailbox   │
     │  / FIFO    │
     └────────────┘
```

- 通过 Mailbox/FIFO 传递消息
- 无需一致性协议
- 编程模型复杂

### 中断（IPI）

```
┌─────────┐  ┌─────────┐
│  Core 0 │  │  Core 1 │
└────┬────┘  └────┬────┘
     │            │
     │  IPI       │
     └────────────┘
```

- 核间中断通知
- 用于同步、调度

## 多核调度

### 对称多处理（SMP）

- 所有核心平等
- OS 动态调度任务
- 共享内存

### 非对称多处理（AMP）

- 核心功能不同（如大小核）
- 固定任务分配
- 可能独立 OS

### 大小核（big.LITTLE）

```
┌─────────┐  ┌─────────┐  ┌─────────┐
│ Big Core│  │ Big Core│  │Little   │
│ (A78)   │  │ (A78)   │  │(A55)    │
└────┬────┘  └────┬────┘  └────┬────┘
     │            │            │
     └────────────┼────────────┘
                  │
           ┌──────▼──────┐
           │  Shared L3  │
           └─────────────┘
```

- **Big 核**：高性能，高功耗
- **Little 核**：低性能，低功耗
- **调度策略**：根据负载动态迁移

## 一致性互联协议

### AMBA CHI（Coherent Hub Interface）

| 组件 | 角色 | 说明 |
|------|------|------|
| RN-F | Request Node (Full) | CPU 核心，发起一致性请求 |
| RN-I | Request Node (I/O) | I/O 设备，发起非一致性请求 |
| HN-F | Home Node (Full) | 一致性目录，管理一致性 |
| HN-I | Home Node (I/O) | I/O 一致性节点 |
| SN-F | Slave Node (Full) | 内存控制器 |
| SN-I | Slave Node (I/O) | I/O 从节点 |

### TileLink

| 级别 | 说明 | 特性 |
|------|------|------|
| TL-UL | Uncached Lightweight | 无缓存，简单 |
| TL-UH | Uncached Heavyweight | 无缓存，支持原子 |
| TL-C | Cached | 支持缓存一致性 |

## 设计注意事项

### RTL 实现要点

1. **一致性协议**：状态机复杂，需仔细验证
2. **互联带宽**：需满足多核并发访问
3. **死锁避免**：NoC 路由需避免死锁
4. **功耗管理**：核心可独立开关
5. **调试**：多核调试复杂

### 面积/功耗权衡

| 配置 | 面积 | 功耗 | 性能 |
|------|------|------|------|
| 2 核 | 2×单核 + 互联 | 2×单核 | ~1.8×单核 |
| 4 核 | 4×单核 + 互联 | 4×单核 | ~3.5×单核 |
| 8 核 | 8×单核 + NoC | 8×单核 | ~6×单核 |
| 16 核 | 16×单核 + NoC | 16×单核 | ~10×单核 |

### 常见陷阱

1. **伪共享（False Sharing）**：不同变量在同一缓存行
2. **一致性延迟**：一致性协议增加内存访问延迟
3. **死锁**：NoC 路由/一致性协议可能死锁
4. **功耗**：多核功耗管理复杂
5. **调试**：多核调试工具和方法
