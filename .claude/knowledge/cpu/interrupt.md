# 中断控制器设计

> **用途**：处理器中断控制器架构设计参考，供 CPU/SoC 中断子系统设计时检索
> **典型应用**：所有 CPU/SoC 设计

---

## 概述

中断控制器负责管理和分发中断请求到处理器核心。

### 中断类型

| 类型 | 说明 | 典型来源 |
|------|------|----------|
| 外部中断 | 来自外部设备 | UART、Timer、GPIO |
| 内部中断 | 来自处理器内部 | Timer、PMU、看门狗 |
| 软件中断 | 软件触发 | 核间通信、系统调用 |
| 异常 | 同步事件 | 未定义指令、页故障 |

---

## RISC-V 中断架构

### PLIC（平台级中断控制器）

#### 中断类型

| 类型 | ID 范围 | 说明 |
|------|---------|------|
| SGI | 0-15 | 软件触发中断 |
| 外部中断 | 1-1023 | 来自外部设备 |

#### 优先级

- 每个中断源可配置优先级（0-7）
- 0 表示禁止中断
- 7 为最高优先级

#### 上下文（Context）

- 每个核心的每个特权级是一个上下文
- 例如：Core 0 的 M-mode 和 S-mode 是两个上下文

#### PLIC 寄存器

| 寄存器 | 偏移 | 说明 |
|--------|------|------|
| priority | 0x000000-0x000FFC | 中断源优先级 |
| pending | 0x001000 | 中断 pending 位 |
| enable | 0x002000 | 中断使能位 |
| threshold | 0x200000 | 优先级阈值 |
| claim/complete | 0x200004 | 中断认领/完成 |

### CLIC（核心级中断控制器）

#### 特性

- **向量中断**：每个中断独立向量地址
- **抢占支持**：高优先级中断可抢占低优先级
- **电平/边沿**：每个中断可配置触发方式
- **最小延迟**：硬件自动保存/恢复上下文

#### CLIC 寄存器

| 寄存器 | 说明 |
|--------|------|
| mintthresh | 中断阈值 |
| mnxti | 下一个中断地址 |
| mcause | 中断原因 |

## ARM GICv3/v4

### 组件

| 组件 | 功能 | 接口 |
|------|------|------|
| Distributor | 全局中断分发 | MMIO |
| Redistributor | 每 CPU 中断路由 | MMIO |
| CPU Interface | CPU 中断应答 | 系统寄存器 |

### 中断类型

| 类型 | ID 范围 | 说明 |
|------|---------|------|
| SGI | 0-15 | 软件触发中断（核间通信） |
| PPI | 16-31 | 私有外设中断（每 CPU） |
| SPI | 32-1019 | 共享外设中断 |
| LPI | 8192+ | 局部外设中断（GICv3） |

### 中断路由

```
┌─────────────────────────────────────────┐
│  Distributor                            │
│  ┌─────────────┐  ┌─────────────┐       │
│  │  SPI 32     │  │  SPI 33     │       │
│  │  → Core 0   │  │  → Core 1   │       │
│  └─────────────┘  └─────────────┘       │
└─────────────────────────────────────────┘
```

- SPI 可路由到任意核心
- PPI 固定到特定核心
- SGI 用于核间通信

### 优先级与抢占

- 256 个优先级（0-255）
- 优先级分组（Group 0/1）
- 支持中断抢占

## 中断处理流程

### 1. 中断触发

```
设备 → 中断线 → 中断控制器 → CPU
```

### 2. 中断应答

```
CPU 读取中断号 → 中断控制器返回最高优先级中断号
```

### 3. 上下文保存

```
CPU 保存 PC、状态寄存器 → 跳转到中断向量表
```

### 4. 中断服务

```
执行中断服务程序（ISR）
```

### 5. 中断返回

```
恢复上下文 → 返回被中断的程序
```

### 6. 中断完成

```
CPU 写中断完成寄存器 → 中断控制器清除 pending
```

## 中断向量表

### RISC-V

```
基地址 = mtvec (M-mode) 或 stvec (S-mode)

直接模式：
  所有中断/异常跳转到基地址

向量模式：
  中断跳转到 基地址 + 4 × 中断号
```

### ARM

```
基地址 = VBAR_ELx

偏移：
  0x000: 同步异常
  0x080: IRQ
  0x100: FIQ
  0x180: SError
```

## 中断优先级

### 固定优先级

- 每个中断源有固定优先级
- 硬件简单
- 不灵活

### 可编程优先级

- 每个中断源可配置优先级
- 灵活
- 需要寄存器配置

### 优先级编码器

```verilog
// 4 输入优先级编码器
module priority_encoder (
    input  wire [3:0] req,
    output reg  [1:0] grant,
    output wire       valid
);
    always @(*) begin
        if (req[3])      grant = 2'd3;
        else if (req[2]) grant = 2'd2;
        else if (req[1]) grant = 2'd1;
        else if (req[0]) grant = 2'd0;
        else             grant = 2'd0;
    end
    assign valid = |req;
endmodule
```

## 中断屏蔽

### 全局中断屏蔽

- 通过状态寄存器全局使能/禁止中断
- RISC-V：mstatus.MIE
- ARM：DAIF 寄存器

### 单个中断屏蔽

- 通过中断控制器使能/禁止每个中断
- PLIC：enable 寄存器
- GIC：GICD_ISENABLERn

### 优先级阈值

- 低于阈值的中断被屏蔽
- PLIC：threshold 寄存器
- GIC：ICC_PMR_EL1

## 核间中断（IPI）

### RISC-V SGI

```
Core 0 写 PLIC → PLIC 触发 Core 1 的 SGI → Core 1 响应
```

### ARM SGI

```
Core 0 写 GICD_SGIR → GIC 触发 Core 1 的 SGI → Core 1 响应
```

### 用途

- 任务调度通知
- TLB 刷新通知
- 缓存一致性通知

## 设计注意事项

### RTL 实现要点

1. **优先级仲裁**：需单周期完成
2. **pending 寄存器**：需原子操作
3. **上下文切换**：需保存/恢复中断状态
4. **死锁避免**：中断处理中避免等待共享资源
5. **功耗**：未使用中断可关闭

### 面积/功耗权衡

| 配置 | 面积 (kGates) | 功耗 | 说明 |
|------|---------------|------|------|
| 简单 8 中断 | 1-2 | 低 | 嵌入式 |
| PLIC 32 中断 | 5-10 | 中 | RISC-V SoC |
| GICv3 1024 中断 | 20-50 | 中高 | ARM SoC |

### 常见陷阱

1. **中断丢失**：pending 位被覆盖
2. **优先级反转**：低优先级中断阻塞高优先级
3. **死锁**：中断处理中等待共享资源
4. **延迟**：中断响应延迟影响实时性
5. **调试**：中断调试困难
