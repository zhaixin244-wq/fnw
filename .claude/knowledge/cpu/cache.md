# 缓存系统设计

> **用途**：处理器缓存层次结构设计参考，供 CPU/SoC 微架构设计时检索
> **典型应用**：所有 CPU 设计

---

## 概述

缓存是处理器性能的关键组件，通过存储最近访问的数据减少内存访问延迟。

### 缓存层次

```
┌─────────────┐
│  CPU Core   │
└──────┬──────┘
       │
┌──────▼──────┐
│  L1 Cache   │  ← 最快，最小（32-64KB）
│  (I/D分离)  │
└──────┬──────┘
       │
┌──────▼──────┐
│  L2 Cache   │  ← 中等（256KB-1MB）
└──────┬──────┘
       │
┌──────▼──────┐
│  L3 Cache   │  ← 最慢，最大（4-64MB）
│  (共享)     │
└──────┬──────┘
       │
┌──────▼──────┐
│  主存 (DDR) │
└─────────────┘
```

### 典型参数

| 级别 | 容量 | 延迟 | 关联度 | 行大小 |
|------|------|------|--------|--------|
| L1 I | 32-64KB | 1-2 周期 | 4-8 路 | 64B |
| L1 D | 32-64KB | 1-2 周期 | 4-8 路 | 64B |
| L2 | 256KB-1MB | 4-10 周期 | 8-16 路 | 64B |
| L3 | 4-64MB | 10-30 周期 | 16-32 路 | 64B |

## 缓存组织

### 1. 直接映射（Direct Mapped）

```
地址：[Tag | Index | Offset]

┌─────────────────────────────────────┐
│  Cache 行                           │
│  ┌─────────┬─────────┬───────────┐  │
│  │  Index  │   Tag   │   Data    │  │
│  ├─────────┼─────────┼───────────┤  │
│  │    0    │  Tag_0  │  Data_0   │  │
│  │    1    │  Tag_1  │  Data_1   │  │
│  │   ...   │   ...   │   ...    │  │
│  └─────────┴─────────┴───────────┘  │
└─────────────────────────────────────┘
```

**优点**：硬件简单，查找快
**缺点**：冲突率高

### 2. 全相联（Fully Associative）

```
地址：[Tag | Offset]

┌─────────────────────────────────────┐
│  Cache 行                           │
│  ┌─────────┬─────────┬───────────┐  │
│  │   Tag   │  Valid  │   Data    │  │
│  ├─────────┼─────────┼───────────┤  │
│  │  Tag_0  │    1    │  Data_0   │  │
│  │  Tag_1  │    1    │  Data_1   │  │
│  │   ...   │   ...   │   ...    │  │
│  └─────────┴─────────┴───────────┘  │
└─────────────────────────────────────┘
```

**优点**：冲突率最低
**缺点**：硬件复杂，需比较所有行

### 3. 组相联（Set Associative）

```
地址：[Tag | Index | Offset]

┌─────────────────────────────────────────────────────┐
│  Set (Index)                                        │
│  ┌──────────────────────────────────────────────┐   │
│  │  Way 0  │  Tag_0  │  Valid │  Data_0        │   │
│  │  Way 1  │  Tag_1  │  Valid │  Data_1        │   │
│  │  Way 2  │  Tag_2  │  Valid │  Data_2        │   │
│  │  Way 3  │  Tag_3  │  Valid │  Data_3        │   │
│  └──────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

**优点**：平衡冲突率和硬件复杂度
**常用**：4-16 路组相联

## 缓存操作

### 读操作

1. **解析地址**：Tag / Index / Offset
2. **查找**：用 Index 定位 Set，比较所有 Way 的 Tag
3. **命中**：返回对应 Way 的 Data[Offset]
4. **未命中**：
   - 从下级缓存/内存取数据
   - 选择替换行（LRU/随机/FIFO）
   - 写入新数据，更新 Tag/Valid

### 写操作

#### 写直达（Write-Through）

```
CPU 写入 → 同时写 Cache 和下级存储
```

- **优点**：一致性简单
- **缺点**：写延迟高，总线流量大

#### 写回（Write-Back）

```
CPU 写入 → 只写 Cache，设置 Dirty 位
替换时 → 如果 Dirty，写回下级存储
```

- **优点**：写延迟低，总线流量小
- **缺点**：一致性复杂，需 Dirty 位

### 写分配（Write Allocate）

- **写分配**：写未命中时，先取数据到 Cache，再写入
- **写不分配**：写未命中时，直接写入下级存储

**常见组合**：
- 写直达 + 写不分配
- 写回 + 写分配

## 替换策略

### LRU（Least Recently Used）

```
4 路 LRU 状态：

访问 Way 0: [0,1,2,3] → [0,1,2,3] (不变)
访问 Way 2: [0,1,2,3] → [2,0,1,3]
访问 Way 1: [2,0,1,3] → [1,2,0,3]
替换：选择 Way 3（最久未使用）
```

**优点**：命中率高
**缺点**：硬件开销大（需维护访问顺序）

### Pseudo-LRU

```
4 路 Pseudo-LRU 树：

        [根]
       /    \
     [左]   [右]
     / \    / \
   W0  W1  W2  W3

访问 Way 0: 左→0, 右→1, 左→0
替换：沿树向下，选择标记为"最久"的路径
```

**优点**：硬件简单
**缺点**：准确率略低于 LRU

### 随机替换

- 随机选择一行替换
- 硬件最简单
- 命中率较低

### FIFO（先进先出）

- 选择最早进入的行替换
- 硬件简单
- 命中率介于 LRU 和随机之间

## 写缓冲（Write Buffer）

### 写直达写缓冲

```
CPU 写入 → [写缓冲] → 下级存储

写缓冲结构：
┌─────────────┐
│  Write Buffer│
│  ┌─────────┐│
│  │ Addr    ││
│  │ Data    ││
│  │ Valid   ││
│  └─────────┘│
└─────────────┘
```

- 缓冲写操作，减少 CPU 等待
- 深度通常 4-16 条目
- 满时需 Stall

---

## 非阻塞缓存（Non-Blocking Cache）

### 原理

- 允许缓存在未命中时继续服务后续请求
- 支持多个未命中同时处理（Miss Status Holding Registers, MSHR）

### MSHR 结构

```
┌─────────────────────────────────────────┐
│  MSHR (Miss Status Holding Register)    │
│  ┌─────────┬─────────┬─────────┬─────┐  │
│  │  Addr   │  State  │  Next   │ ... │  │
│  ├─────────┼─────────┼─────────┼─────┤  │
│  │ 0x1000  │ Fill    │  NULL   │ ... │  │
│  │ 0x2000  │ Wait    │ 0x2004  │ ... │  │
│  └─────────┴─────────┴─────────┴─────┘  │
└─────────────────────────────────────────┘
```

- 每个 MSHR 跟踪一个未命中
- 相同地址的后续访问合并到同一 MSHR
- 减少未命中惩罚

## 缓存一致性

### 问题

多核系统中，每个核心有私有缓存，同一地址可能在多个缓存中有不同值。

### 监听协议（Snooping）

```
┌─────────┐  ┌─────────┐  ┌─────────┐
│  Core 0 │  │  Core 1 │  │  Core 2 │
│  Cache  │  │  Cache  │  │  Cache  │
└────┬────┘  └────┬────┘  └────┬────┘
     │            │            │
     └────────────┼────────────┘
                  │
           ┌──────▼──────┐
           │   总线/互联  │
           └─────────────┘
```

#### MESI 协议

| 状态 | 含义 | 说明 |
|------|------|------|
| M (Modified) | 已修改 | 独占且脏，需写回 |
| E (Exclusive) | 独占 | 独占且干净 |
| S (Shared) | 共享 | 多核共享 |
| I (Invalid) | 无效 | 数据无效 |

#### MOESI 协议

- 增加 O (Owned) 状态
- O 状态：数据被修改，但其他核可能有共享副本
- 减少写回操作

### 目录协议（Directory）

```
┌─────────┐  ┌─────────┐  ┌─────────┐
│  Core 0 │  │  Core 1 │  │  Core 2 │
│  Cache  │  │  Cache  │  │  Cache  │
└────┬────┘  └────┬────┘  └────┬────┘
     │            │            │
     └────────────┼────────────┘
                  │
           ┌──────▼──────┐
           │   Directory │
           │   (共享内存) │
           └─────────────┘
```

- 每个内存块维护目录条目
- 记录哪些缓存有该块的副本
- 点对点通信，减少总线流量

## 虚拟缓存 vs 物理缓存

### 虚拟缓存（VIVT）

```
CPU 地址 (VA) → 缓存查找 → 数据
```

- **优点**：无需 TLB，延迟低
- **缺点**：同义问题（多个 VA 映射同一 PA）

### 物理缓存（PIPT）

```
CPU 地址 (VA) → TLB → PA → 缓存查找 → 数据
```

- **优点**：无同义问题
- **缺点**：需 TLB，延迟高

### 虚拟索引物理标签（VIPT）

```
CPU 地址 (VA) → [Index 部分直接用 VA] → 缓存查找
                → [Tag 部分用 PA]
```

- **优点**：TLB 和缓存并行查找
- **限制**：Index 位数不能超过页内偏移位数

## 预取（Prefetch）

### 硬件预取

| 类型 | 说明 | 触发条件 |
|------|------|----------|
| 顺序预取 | 预取下一行 | 访问当前行 |
| 步长预取 | 检测固定步长模式 | 连续相同步长 |
| 流预取 | 检测流访问模式 | 连续访问序列 |

### 软件预取

- 编译器插入预取指令
- 程序员显式预取
- 需要 ISA 支持预取指令

## 设计注意事项

### RTL 实现要点

1. **Tag 比较**：多路并行比较，单周期完成
2. **数据选择**：多路数据 MUX，需考虑时序
3. **替换逻辑**：LRU 需维护状态，面积开销
4. **写缓冲**：深度需根据写频率设计
5. **一致性**：多核需一致性协议

### 面积/功耗权衡

| 配置 | 面积 (KB) | 功耗 | 命中率 |
|------|----------|------|--------|
| L1 32KB/4-way | 32+Tag | 低 | ~95% |
| L1 64KB/8-way | 64+Tag | 中 | ~97% |
| L2 256KB/8-way | 256+Tag | 中高 | ~99% |
| L2 1MB/16-way | 1024+Tag | 高 | ~99.5% |

### 常见陷阱

1. **别名问题**：虚拟缓存需处理同义问题
2. **一致性延迟**：多核一致性协议增加延迟
3. **预取污染**：无效预取浪费缓存空间
4. **替换策略**：LRU 面积大，Pseudo-LRU 是折中
5. **写缓冲满**：写直达写缓冲满时需 Stall
