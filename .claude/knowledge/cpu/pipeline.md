# 处理器流水线设计

> **用途**：处理器流水线架构设计参考，供 CPU 微架构设计时检索
> **典型应用**：所有 CPU 设计

---

## 概述

流水线是处理器性能提升的核心技术，将指令执行分解为多个阶段，每阶段并行处理不同指令。

### 流水线优势

- **提高吞吐**：每个周期完成一条指令（理想情况）
- **提高频率**：每级组合逻辑减少，时钟频率提升
- **资源复用**：同一硬件在不同时钟周期服务不同指令

---

## 经典五级流水线

```
IF      ID      EX      MEM     WB
取指    译码    执行    访存    写回
|       |       |       |       |
|<---------- 1 条指令 ---------->|
        |<---------- 1 条指令 ---------->|
                |<---------- 1 条指令 ---------->|
```

### 各阶段功能

| 阶段 | 功能 | 关键硬件 |
|------|------|----------|
| IF | 从 I-Cache 取指令 | PC 寄存器、I-Cache、分支预测器 |
| ID | 指令译码、读寄存器 | 译码器、寄存器堆 |
| EX | ALU 运算、地址计算 | ALU、乘法器、移位器 |
| MEM | 访问 D-Cache | D-Cache、Load/Store 单元 |
| WB | 结果写回寄存器 | 寄存器堆写端口 |

### 流水线寄存器

```
IF/ID: 指令字、PC
ID/EX: 译码结果、寄存器值、立即数、控制信号
EX/MEM: ALU 结果、存储数据、控制信号
MEM/WB: 访存结果、ALU 结果、控制信号
```

## 流水线冒险

### 1. 数据冒险（Data Hazard）

**RAW（Read After Write）**：后续指令需要前序指令的结果

```
ADD R1, R2, R3    ; 写 R1
SUB R4, R1, R5    ; 读 R1 → 冒险！
```

**解决方案**：

| 方案 | 延迟 | 硬件复杂度 | 说明 |
|------|------|-----------|------|
| 插入气泡（Stall） | 2-3 周期 | 低 | 暂停流水线等待数据 |
| 数据前递（Forwarding） | 0 周期 | 中 | 将结果直接旁路到需要的阶段 |
| 编译器调度 | 0 周期 | 无 | 编译器重排指令避免冒险 |
| 乱序执行 | 0 周期 | 高 | 动态调度，Tomasulo 算法 |

**前递路径**：

```
EX/MEM → EX（EX 结果前递）
MEM/WB → EX（MEM 结果前递）
MEM/WB → MEM（Store 数据前递）
```

### 2. 控制冒险（Control Hazard）

**分支指令**：分支目标地址需到 EX/MEM 阶段才确定

```
BEQ R1, R2, label   ; 分支判断
ADD R3, R4, R5      ; 可能不该执行
SUB R6, R7, R8      ; 可能不该执行
```

**解决方案**：

| 方案 | 惩罚周期 | 准确率 | 说明 |
|------|----------|--------|------|
| 固定预测（不跳转） | 2-3 | ~50% | 最简单 |
| 静态预测（后向跳转） | 2-3 | ~60% | 循环友好 |
| 1-bit BHT | 1-2 | ~85% | 简单历史 |
| 2-bit BHT | 0-1 | ~90% | 饱和计数器 |
| BTB + BHT | 0-1 | ~95% | 缓存分支目标 |
| TAGE | 0 | ~97% | 多级历史预测 |

### 3. 结构冒险（Structural Hazard）

**资源冲突**：多条指令同时需要同一硬件资源

```
LOAD R1, [R2]   ; 需要 D-Cache
ADD R3, R4, R5  ; 需要 ALU
```

**解决方案**：

- **资源复制**：分离 I-Cache 和 D-Cache（哈佛架构）
- **流水化**：将资源流水化，每周期可接受新请求
- **仲裁**：冲突时优先级仲裁，低优先级指令 Stall

## 超标量流水线

### 双发射流水线

```
IF      ID      EX      MEM     WB
取指    译码    执行    访存    写回
|       |       |       |       |
|<---------- 2 条指令 ---------->|
        |<---------- 2 条指令 ---------->|
                |<---------- 2 条指令 ---------->|
```

### 超标量设计要点

| 要素 | 说明 |
|------|------|
| 取指宽度 | 每周期取多条指令（需 I-Cache 多端口） |
| 译码宽度 | 多个译码器并行 |
| 发射策略 | 顺序发射 / 乱序发射 |
| 执行单元 | 多个 ALU、Load/Store 单元 |
| 写回端口 | 多端口寄存器堆 |

### 发射策略对比

| 策略 | 说明 | 优势 | 劣势 |
|------|------|------|------|
| 顺序发射顺序完成 | 最简单 | 硬件简单 | 性能受限 |
| 顺序发射乱序完成 | 中等复杂 | 兼顾性能和复杂度 | 需重排序缓冲 |
| 乱序发射乱序完成 | 最复杂 | 性能最高 | 硬件最复杂 |

## 乱序执行

### Tomasulo 算法

```
┌─────────────┐
│   指令队列   │
└──────┬──────┘
       │
┌──────▼──────┐
│  寄存器重命名 │  ← 消除 WAR/WAW 冒险
└──────┬──────┘
       │
┌──────▼──────┐
│ 保留站 (RS)  │  ← 存储操作数和操作
└──────┬──────┘
       │
┌──────▼──────┐
│  执行单元    │  ← 多个功能单元并行
└──────┬──────┘
       │
┌──────▼──────┐
│  CDB 总线   │  ← 公共数据总线广播结果
└──────┬──────┘
       │
┌──────▼──────┐
│ 重排序缓冲   │  ← 保证顺序提交
└──────┬──────┘
       │
┌──────▼──────┐
│  寄存器堆    │
└─────────────┘
```

### 关键组件

| 组件 | 功能 | 硬件开销 |
|------|------|----------|
| 保留站 (RS) | 存储待执行指令和操作数 | 大 |
| 重排序缓冲 (ROB) | 保证顺序提交 | 大 |
| 寄存器重命名 | 消除假依赖 | 中 |
| CDB 总线 | 广播执行结果 | 中 |
| 加载/存储队列 | 管理内存访问顺序 | 中 |

## 分支预测

### BTB（分支目标缓冲）

```
┌─────────────────────────────────────┐
│  BTB (Branch Target Buffer)         │
│  ┌─────────┬─────────┬─────────┐   │
│  │  Tag    │  Target │  Valid  │   │
│  ├─────────┼─────────┼─────────┤   │
│  │ 0x1000  │ 0x2000  │    1    │   │
│  │ 0x1004  │ 0x3000  │    1    │   │
│  │  ...    │  ...    │   ...   │   │
│  └─────────┴─────────┴─────────┘   │
└─────────────────────────────────────┘
```

### BHT（分支历史表）

```
2-bit 饱和计数器：

  强不跳转  弱不跳转  弱跳转  强跳转
    00        01       10      11
     │         │        │       │
     ├─────────┼────────┼───────┤
     │  预测不跳转  │  预测跳转  │
```

### TAGE（Tagged Geometric History）

- 多级历史长度：L1, L2, L4, L8, L16, ...
- 每级独立预测表
- 最长匹配优先
- 准确率 ~97%

## 设计注意事项

### RTL 实现要点

1. **流水线控制**：每级需 valid 信号，Stall/Flush 逻辑
2. **前递逻辑**：EX/MEM → EX，MEM/WB → EX
3. **冒险检测**：RAW 检测、Load-Use 检测
4. **分支处理**：预测失败时 Flush 流水线
5. **异常处理**：精确异常需 ROB 或重排序

### 面积/功耗权衡

| 流水线类型 | 面积 (kGates) | 功耗 | 性能 (IPC) |
|-----------|---------------|------|-----------|
| 5 级顺序 | 10-20 | 低 | 0.5-0.8 |
| 8 级顺序 | 20-40 | 中 | 0.7-1.0 |
| 10+ 级乱序 | 50-100+ | 高 | 1.5-3.0 |
| 超标量乱序 | 100-300+ | 很高 | 2.0-5.0 |

### 常见陷阱

1. **Load-Use 冒险**：Load 后立即使用需 Stall 1 周期
2. **分支预测失败**：需 Flush 流水线，惩罚大
3. **精确异常**：乱序执行需 ROB 保证精确异常
4. **前递优先级**：多个前递路径需仲裁
5. **时序瓶颈**：前递路径可能成为关键路径
