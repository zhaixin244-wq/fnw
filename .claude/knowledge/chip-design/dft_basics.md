# DFT（可测试性设计）基础

> **用途**：DFT 基础概念参考，供芯片测试设计时检索
> **典型应用**：所有芯片 DFT 设计

---

## 概述

DFT（Design for Testability）是提高芯片可测试性的设计技术。

### DFT 目标

- **可控性**：能够控制内部节点状态
- **可观测性**：能够观测内部节点状态
- **故障覆盖率**：检测尽可能多的故障
- **测试时间**：最小化测试时间

### DFT 技术

| 技术 | 说明 | 应用 |
|------|------|------|
| 扫描链 | 串行测试路径 | 逻辑测试 |
| BIST | 内建自测试 | 存储器/逻辑测试 |
| 边界扫描 | JTAG 接口 | 板级测试 |
| ATPG | 自动测试向量生成 | 测试向量生成 |

## 扫描链

### 扫描链原理

```
扫描链结构：
┌─────────────────────────────────────────────┐
│  Scan In                                    │
│     │                                       │
│     ▼                                       │
│  ┌─────┐  ┌─────┐  ┌─────┐  ┌─────┐       │
│  │ FF1 │→│ FF2 │→│ FF3 │→│ FF4 │       │
│  └─────┘  └─────┘  └─────┘  └─────┘       │
│     │                                       │
│     ▼                                       │
│  Scan Out                                   │
└─────────────────────────────────────────────┘
```

### 扫描单元

```
扫描触发器（Scan FF）：
┌─────────────────────────────────────────────┐
│  ┌─────────────────────────────────────┐    │
│  │  MUX                                │    │
│  │  ┌─────┐                            │    │
│  │  │     │ D ← 数据输入               │    │
│  │  │  MUX├───→ FF → Q                 │    │
│  │  │     │ SI ← 扫描输入              │    │
│  │  └─────┘                            │    │
│  │     ↑                               │    │
│  │    SE (扫描使能)                     │    │
│  └─────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
```

### 扫描模式

| 模式 | SE | 功能 |
|------|----|------|
| 正常模式 | 0 | 正常功能 |
| 扫描模式 | 1 | 扫描移位 |

### 扫描链插入

```tcl
# Synopsys DFT Compiler
set_scan_configuration -style multiplexed
insert_scan

# 报告扫描链
report_scan_chain > scan_chain.rpt
```

## ATPG（自动测试向量生成）

### ATPG 流程

```
门级网表
    ↓
故障模型生成
    ↓
测试向量生成
    ↓
故障仿真
    ↓
覆盖率报告
```

### 故障模型

| 故障模型 | 说明 | 应用 |
|----------|------|------|
| 固定故障 | 节点固定 0/1 | 基本测试 |
| 跳变故障 | 节点 0→1/1→0 | 延迟测试 |
| 桥接故障 | 节点间短路 | 互连测试 |
| 路径延迟故障 | 路径延迟过大 | 时序测试 |

### ATPG 向量

```
测试向量结构：
┌─────────────────────────────────────────────┐
│  Scan In 数据                               │
│  ─────────────────────────────────────────  │
│  1 0 1 1 0 0 1 0 1 1 ...                   │
└─────────────────────────────────────────────┘
┌─────────────────────────────────────────────┐
│  测试周期                                    │
│  ─────────────────────────────────────────  │
│  Capture Clock                              │
└─────────────────────────────────────────────┘
┌─────────────────────────────────────────────┐
│  Scan Out 数据                              │
│  ─────────────────────────────────────────  │
│  1 0 1 0 1 1 0 0 1 0 ...                   │
└─────────────────────────────────────────────┘
```

### ATPG 工具

```tcl
# Synopsys TetraMAX
read_netlist netlist.v
read_cell_library library.lib
run_atpg

# 报告覆盖率
report_fault_coverage > coverage.rpt

# 输出测试向量
write_patterns -format stil > patterns.stil
```

## 边界扫描（JTAG）

### JTAG 接口

| 信号 | 方向 | 说明 |
|------|------|------|
| TCK | I | 测试时钟 |
| TMS | I | 测试模式选择 |
| TDI | I | 测试数据输入 |
| TDO | O | 测试数据输出 |
| TRST | I | 测试复位（可选） |

### JTAG 状态机

```
JTAG TAP 状态机：
┌─────────────────────────────────────────────┐
│  Test-Logic-Reset                           │
│     │                                       │
│     ▼                                       │
│  Run-Test/Idle                              │
│     │                                       │
│     ▼                                       │
│  Select-DR ←→ Select-IR                    │
│     │              │                        │
│     ▼              ▼                        │
│  Capture-DR    Capture-IR                  │
│     │              │                        │
│     ▼              ▼                        │
│  Shift-DR      Shift-IR                    │
│     │              │                        │
│     ▼              ▼                        │
│  Exit1-DR      Exit1-IR                    │
│     │              │                        │
│     ▼              ▼                        │
│  Pause-DR      Pause-IR                    │
│     │              │                        │
│     ▼              ▼                        │
│  Exit2-DR      Exit2-IR                    │
│     │              │                        │
│     ▼              ▼                        │
│  Update-DR     Update-IR                   │
└─────────────────────────────────────────────┘
```

### JTAG 寄存器

| 寄存器 | 说明 | 用途 |
|--------|------|------|
| IR | 指令寄存器 | 选择测试功能 |
| DR | 数据寄存器 | 测试数据 |
| BSR | 边界扫描寄存器 | I/O 测试 |
| IDCODE | 设备 ID | 设备识别 |

## 测试压缩

### 测试压缩目的

- **减少测试数据量**：降低测试成本
- **减少测试时间**：提高测试效率
- **降低测试功耗**：减少测试功耗

### 测试压缩技术

| 技术 | 说明 | 工具 |
|------|------|------|
| 编码压缩 | 编码测试数据 | Synopsys DFTMAX |
| 广播扫描 | 共享扫描链 | Cadence Modus |
| 虚拟扫描 | 虚拟扫描链 | Synopsys DFTMAX |

### 测试压缩流程

```tcl
# Synopsys DFTMAX
set_dft_configuration -compression enable
insert_dft

# 报告压缩
report_compression > compression.rpt
```

## 故障覆盖率

### 覆盖率定义

```
故障覆盖率 = 检测到的故障数 / 总故障数 × 100%

目标：> 98%
```

### 覆盖率报告

```tcl
# TetraMAX 覆盖率报告
report_fault_coverage > coverage.rpt

报告内容：
- 总故障数
- 检测到的故障数
- 未检测到的故障数
- 故障覆盖率
```

### 提高覆盖率

1. **增加测试向量**：生成更多向量
2. **插入测试点**：增加可控/可观测点
3. **优化扫描链**：改善扫描链质量
4. **使用 BIST**：补充 ATPG 测试
