# 虚拟化

> **用途**：处理器虚拟化机制设计参考，供虚拟化平台/SoC 设计时检索
> **典型应用**：虚拟化平台、服务器、DPU

---

## 概述

虚拟化允许在一个物理硬件上运行多个虚拟机（VM），每个 VM 有独立的操作系统和内存空间。

### 虚拟化类型

| 类型 | 说明 | 典型应用 |
|------|------|----------|
| 全虚拟化 | 完全模拟硬件 | 传统虚拟化 |
| 半虚拟化 | 修改 Guest OS | Xen |
| 硬件辅助 | 硬件支持虚拟化 | 现代虚拟化 |

## 两阶段地址转换

### 为什么需要两阶段

- **Guest OS**：管理 Guest VA → Guest PA
- **Hypervisor**：管理 Guest PA → Host PA

### ARM 两阶段转换

```
┌─────────────────────────────────────────────┐
│  Stage 1: Guest OS 页表                     │
│  Guest VA → Guest PA                        │
│  (Guest OS 管理)                            │
└──────────────────┬──────────────────────────┘
                   │
┌──────────────────▼──────────────────────────┐
│  Stage 2: Hypervisor 页表                   │
│  Guest PA → Host PA                         │
│  (Hypervisor 管理)                          │
└─────────────────────────────────────────────┘
```

### RISC-V 两阶段转换

```
┌─────────────────────────────────────────────┐
│  VS-Stage: Guest OS 页表                    │
│  Guest VA → Guest PA                        │
│  (Guest OS 管理)                            │
└──────────────────┬──────────────────────────┘
                   │
┌──────────────────▼──────────────────────────┐
│  G-Stage: Hypervisor 页表                   │
│  Guest PA → Host PA                         │
│  (Hypervisor 管理)                          │
└─────────────────────────────────────────────┘
```

### x86 EPT（扩展页表）

```
┌─────────────────────────────────────────────┐
│  Guest 页表                                 │
│  Guest VA → Guest PA                        │
│  (Guest OS 管理)                            │
└──────────────────┬──────────────────────────┘
                   │
┌──────────────────▼──────────────────────────┐
│  EPT (Extended Page Table)                  │
│  Guest PA → Host PA                         │
│  (Hypervisor 管理)                          │
└─────────────────────────────────────────────┘
```

## 虚拟化异常

### ARM 异常路由

| 异常类型 | 路由到 | 说明 |
|----------|--------|------|
| 同步异常 | EL1/EL2 | 根据 HCR_EL2 配置 |
| IRQ | EL1/EL2 | 根据 HCR_EL2 配置 |
| FIQ | EL1/EL2 | 根据 HCR_EL2 配置 |
| SError | EL1/EL2 | 根据 HCR_EL2 配置 |

### RISC-V 异常路由

| 异常类型 | 路由到 | 说明 |
|----------|--------|------|
| Guest 异常 | VS-mode | Guest OS 处理 |
| Hypervisor 异常 | HS-mode | Hypervisor 处理 |
| 两阶段异常 | HS-mode | 需要 Hypervisor 介入 |

## 设备直通

### 设备直通原理

```
┌─────────────────────────────────────────────┐
│  Guest OS                                   │
│  ┌──────────────────────────────────────┐   │
│  │  设备驱动                            │   │
│  │  (访问 Guest 物理地址)               │   │
│  └──────────────────────────────────────┘   │
├─────────────────────────────────────────────┤
│  Hypervisor                                 │
│  ┌──────────────────────────────────────┐   │
│  │  IOMMU                               │   │
│  │  (Guest PA → Host PA)                │   │
│  └──────────────────────────────────────┘   │
├─────────────────────────────────────────────┤
│  物理设备                                   │
│  ┌──────────────────────────────────────┐   │
│  │  DMA 引擎                            │   │
│  │  (使用 Host PA)                      │   │
│  └──────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

### IOMMU 在设备直通中的作用

- **地址转换**：设备 DMA 地址转换
- **内存保护**：防止设备访问未授权内存
- **设备隔离**：隔离不同设备的内存访问
- **中断重映射**：重映射设备中断

## 虚拟机迁移

### 迁移流程

1. **预迁移**：暂停 VM，保存状态
2. **内存迁移**：复制内存页面到目标主机
3. **设备状态迁移**：保存/恢复设备状态
4. **切换**：在目标主机启动 VM

### 迁移挑战

- **内存一致性**：确保内存页面一致
- **设备状态**：保存/恢复设备状态
- **中断状态**：保存/恢复中断状态
- **时间同步**：保持时间同步

## 嵌套虚拟化

### 嵌套虚拟化原理

```
┌─────────────────────────────────────────────┐
│  L0 Hypervisor                              │
│  (物理机 Hypervisor)                        │
├─────────────────────────────────────────────┤
│  L1 Guest OS                                │
│  (运行在 L0 上的 Guest)                      │
├─────────────────────────────────────────────┤
│  L2 Guest OS                                │
│  (运行在 L1 上的 Guest)                      │
└─────────────────────────────────────────────┘
```

### 嵌套虚拟化挑战

- **多级地址转换**：需要多级页表
- **性能开销**：多级转换增加延迟
- **复杂性**：异常处理更复杂

## 虚拟化性能优化

### 硬件加速

| 技术 | 说明 | 优势 |
|------|------|------|
| 两阶段转换 | 硬件支持地址转换 | 减少 Hypervisor 介入 |
| 设备直通 | 设备直接分配给 VM | 减少虚拟化开销 |
| 中断虚拟化 | 硬件中断虚拟化 | 减少中断处理延迟 |
| 时间虚拟化 | 硬件时间虚拟化 | 提高时间精度 |

### 软件优化

| 技术 | 说明 | 优势 |
|------|------|------|
| 内存气球 | 动态调整 VM 内存 | 提高内存利用率 |
| KSM | 内核相同页面合并 | 减少内存使用 |
| 大页 | 使用大页 | 减少 TLB 压力 |
| NUMA 感知 | NUMA 亲和性 | 提高内存访问性能 |

## 虚拟化安全

### 安全隔离

- **内存隔离**：VM 之间内存隔离
- **设备隔离**：设备访问隔离
- **中断隔离**：中断路由隔离
- **时间隔离**：时间资源隔离

### 安全威胁

| 威胁 | 说明 | 缓解 |
|------|------|------|
| 侧信道攻击 | 通过共享资源泄露信息 | 资源隔离 |
| 逃逸攻击 | 从 Guest 逃逸到 Host | 安全更新 |
| 拒绝服务 | 耗尽 Host 资源 | 资源限制 |

## 设计注意事项

### RTL 实现要点

1. **两阶段转换**：需支持多级页表
2. **异常路由**：需正确路由异常到 Guest/Hypervisor
3. **设备直通**：需支持 IOMMU
4. **性能优化**：需支持硬件加速
5. **安全隔离**：需确保 VM 之间隔离

### 面积/功耗权衡

| 配置 | 面积 | 功耗 | 说明 |
|------|------|------|------|
| 无虚拟化 | 小 | 低 | 嵌入式 |
| 基本虚拟化 | 中 | 中 | 服务器 |
| 完整虚拟化 | 大 | 高 | 云计算 |

### 常见陷阱

1. **性能开销**：两阶段转换增加延迟
2. **复杂性**：虚拟化逻辑复杂
3. **安全隔离**：需确保 VM 之间隔离
4. **设备直通**：IOMMU 配置复杂
5. **调试困难**：虚拟化调试复杂
