# 地址空间

> **用途**：处理器地址空间管理设计参考，供 CPU/SoC 内存子系统设计时检索
> **典型应用**：所有支持虚拟内存的 CPU

---

## 概述

地址空间是处理器可访问的内存范围，包括虚拟地址空间和物理地址空间。

### 虚拟地址空间 (VAS)

- 程序使用的地址
- 每个进程独立
- 通过 MMU 转换为物理地址

### 物理地址空间 (PAS)

- 实际内存地址
- 所有进程共享
- 用于访问缓存和内存

## RISC-V 地址空间

### 虚拟地址空间

| 模式 | 虚拟地址位数 | 地址范围 | 页表级数 |
|------|-------------|---------|----------|
| Sv32 | 32-bit | 0x00000000 - 0xFFFFFFFF | 2 级 |
| Sv39 | 39-bit | 0x000000007F - 0xFFFFFF8000 | 3 级 |
| Sv48 | 48-bit | 0x000000000000 - 0xFFFFFFFFFFFF | 4 级 |
| Sv57 | 57-bit | 0x00000000000000 - 0xFFFFFFFFFFFFFF | 5 级 |

### Sv39 地址布局

```
虚拟地址 (39-bit):
┌─────────┬─────────┬─────────┬─────────┐
│  VPN[2] │  VPN[1] │  VPN[0] │  Offset │
│  9 bit  │  9 bit  │  9 bit  │  12 bit │
└─────────┴─────────┴─────────┴─────────┘

地址范围:
- 低地址: 0x0000000000 - 0x3FFFFFFFFF (用户空间)
- 高地址: 0xFFFFFFC000 - 0xFFFFFFFFFF (内核空间)
```

### Sv48 地址布局

```
虚拟地址 (48-bit):
┌─────────┬─────────┬─────────┬─────────┬─────────┐
│  VPN[3] │  VPN[2] │  VPN[1] │  VPN[0] │  Offset │
│  9 bit  │  9 bit  │  9 bit  │  9 bit  │  12 bit │
└─────────┴─────────┴─────────┴─────────┴─────────┘

地址范围:
- 低地址: 0x000000000000 - 0x7FFFFFFFFFFF (用户空间)
- 高地址: 0xFFFF80000000 - 0xFFFFFFFFFFFF (内核空间)
```

### 物理地址空间

| 实现 | 物理地址位数 | 地址范围 |
|------|-------------|---------|
| Sv32 | 34-bit | 0x000000000 - 0xFFFFFFFFF |
| Sv39 | 56-bit | 0x00000000000000 - 0xFFFFFFFFFFFFFF |
| Sv48 | 56-bit | 0x00000000000000 - 0xFFFFFFFFFFFFFF |

## ARM 地址空间

### AArch64 虚拟地址空间

```
虚拟地址 (48-bit or 52-bit):

EL0/EL1 (用户/内核):
┌─────────────────────────────────────────────┐
│  0x00000000000000 - 0x0000FFFFFFFFFFFF     │  用户空间
│  0xFFFF0000000000 - 0xFFFFFFFFFFFFFFFF     │  内核空间
└─────────────────────────────────────────────┘

EL2 (Hypervisor):
┌─────────────────────────────────────────────┐
│  0x00000000000000 - 0x0000FFFFFFFFFFFF     │  Guest 空间
└─────────────────────────────────────────────┘

EL3 (Secure Monitor):
┌─────────────────────────────────────────────┐
│  0x00000000000000 - 0x0000FFFFFFFFFFFF     │  安全空间
└─────────────────────────────────────────────┘
```

| 实现 | 物理地址位数 | 地址范围 |
|------|-------------|---------|
| 48-bit | 48-bit | 0x000000000000 - 0xFFFFFFFFFFFF |
| 52-bit | 52-bit | 0x0000000000000 - 0xFFFFFFFFFFFFF |

## 地址空间 ID (ASID)

### ASID 作用

- **区分进程**：同一 VPN 不同进程映射不同 PA
- **避免刷新**：上下文切换时无需刷新 TLB
- **提高性能**：减少 TLB 未命中

### ASID 位宽

| 架构 | ASID 位宽 | 地址空间数 |
|------|----------|-----------|
| RISC-V | 8-16 bit | 256-65536 |
| ARM | 8-16 bit | 256-65536 |
| x86 | 12 bit | 4096 |

### ASID 管理

- **静态分配**：每个进程固定 ASID
- **动态分配**：OS 动态分配 ASID
- **ASID 复用**：ASID 用完后复用

## 虚拟化地址空间

### 两阶段地址转换

```
Guest VA → Guest PA → Host PA

Stage 1: Guest VA → Guest PA (Guest OS 管理)
Stage 2: Guest PA → Host PA (Hypervisor 管理)
```

### ARM 两阶段转换

```
┌─────────────────────────────────────────────┐
│  Stage 1: Guest OS 页表                     │
│  Guest VA → Guest PA                        │
└──────────────────┬──────────────────────────┘
                   │
┌──────────────────▼──────────────────────────┐
│  Stage 2: Hypervisor 页表                   │
│  Guest PA → Host PA                         │
└─────────────────────────────────────────────┘
```

### RISC-V 两阶段转换

```
┌─────────────────────────────────────────────┐
│  VS-Stage: Guest OS 页表                    │
│  Guest VA → Guest PA                        │
└──────────────────┬──────────────────────────┘
                   │
┌──────────────────▼──────────────────────────┐
│  G-Stage: Hypervisor 页表                   │
│  Guest PA → Host PA                         │
└─────────────────────────────────────────────┘
```

## 设计注意事项

### RTL 实现要点

1. **地址检查**：需检查地址范围和对齐
2. **权限检查**：需在地址转换时检查权限
3. **ASID 管理**：需支持 ASID 分配和复用
4. **虚拟化**：需支持两阶段地址转换
5. **异常处理**：需处理地址错误、权限故障

### 面积/功耗权衡

| 配置 | 面积 | 功耗 | 说明 |
|------|------|------|------|
| Sv39 | 小 | 低 | 嵌入式/应用处理器 |
| Sv48 | 中 | 中 | 服务器 |
| 4 级 ARM | 中 | 中 | 通用 |

### 常见陷阱

1. **地址空间大小**：需根据应用选择合适的地址空间
2. **ASID 管理**：ASID 有限，需复用
3. **虚拟化开销**：两阶段转换增加延迟
4. **地址对齐**：需检查地址对齐
5. **权限检查**：需在地址转换时完成
