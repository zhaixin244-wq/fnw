## 地址转换对比

| 架构 | 页表级数 | 页大小 | TLB 条目 | 特性 |
|------|----------|--------|----------|------|
| RISC-V Sv32 | 2 级 | 4KB | 32-128 | 简单，适合嵌入式 |
| RISC-V Sv39 | 3 级 | 4KB/2MB/1GB | 64-256 | 39-bit 虚拟地址 |
| RISC-V Sv48 | 4 级 | 4KB/2MB/1GB/512GB | 64-256 | 48-bit 虚拟地址 |
| ARM AArch64 | 3-4 级 | 4KB/16KB/64KB | 128-1024 | 灵活页大小 |
| x86-64 | 4 级 | 4KB/2MB/1GB | 128-1024 | 成熟，广泛使用 |

## 关键概念

### 虚拟地址 (VA)

- 程序使用的地址
- 每个进程独立的地址空间
- 通过 MMU 转换为物理地址

### 物理地址 (PA)

- 实际内存地址
- 所有进程共享
- 用于访问缓存和内存

### 页表 (Page Table)

- 虚拟地址到物理地址的映射表
- 由操作系统管理
- 存储在内存中

### TLB (Translation Lookaside Buffer)

- 页表的缓存
- 加速地址转换
- 硬件管理

## 设计要素

| 要素 | 说明 | 权衡 |
|------|------|------|
| 页表级数 | 级数越多，支持地址空间越大，但遍历延迟越高 | 性能 vs 地址空间 |
| TLB 大小 | 越大命中率越高，但面积和功耗越大 | 面积 vs 命中率 |
| 页大小 | 大页减少 TLB 压力，但内存碎片增加 | 灵活性 vs 效率 |
| 替换策略 | LRU 命中率高但硬件复杂 | 面积 vs 命中率 |

## 典型应用场景

| 场景 | 需求 |
|------|------|
| 嵌入式 MCU | 无 MMU 或简单 PMP |
| 应用处理器 | 完整 MMU + TLB + 虚拟化 |
| 服务器 CPU | 大 TLB + 大页 + 虚拟化 + IOMMU |
| 实时系统 | 确定性延迟、无页故障 |
| 安全 SoC | 内存隔离、TrustZone |
