# TLB（后备转换缓冲）

> **用途**：处理器 TLB 架构设计参考，供 CPU/SoC MMU 设计时检索
> **典型应用**：所有支持虚拟内存的 CPU

---

## 概述

TLB（Translation Lookaside Buffer）是页表的缓存，存储最近使用的虚拟地址到物理地址的映射。

### 为什么需要 TLB

- **页表遍历延迟**：多级页表遍历需多次内存访问（3-4 周期/级）
- **TLB 命中**：1 周期完成地址转换
- **性能影响**：TLB 命中率直接影响系统性能

---

## TLB 结构

### 基本结构

```
┌─────────────────────────────────────────────────────┐
│  TLB                                                │
│  ┌─────────┬─────────┬─────────┬─────────┬───────┐  │
│  │   Tag   │  VPN    │  PPN    │  Attr   │ Valid │  │
│  ├─────────┼─────────┼─────────┼─────────┼───────┤  │
│  │ 0x1234  │ 0x1234  │ 0x5678  │ RWXU    │   1   │  │
│  │ 0x1235  │ 0x1235  │ 0x5679  │ RW-U    │   1   │  │
│  │   ...   │   ...   │   ...   │   ...   │  ...  │  │
│  └─────────┴─────────┴─────────┴─────────┴───────┘  │
└─────────────────────────────────────────────────────┘
```

### TLB 条目字段

| 字段 | 说明 | 位宽 |
|------|------|------|
| Tag | 标签（用于比较） | 20-40 bit |
| VPN | 虚拟页号 | 20-40 bit |
| PPN | 物理页号 | 20-40 bit |
| Attr | 属性（权限、Cache） | 8-16 bit |
| Valid | 有效位 | 1 bit |
| ASID | 地址空间 ID | 8-16 bit |
| G | 全局标志 | 1 bit |

### 属性字段

| 位 | 名称 | 说明 |
|----|------|------|
| R | 可读 | 允许读取 |
| W | 可写 | 允许写入 |
| X | 可执行 | 允许执行 |
| U | 用户模式 | 用户模式可访问 |
| D | 脏 | 已被修改 |
| A | 访问 | 已被访问 |
| SH | 共享性 | 内部/外部共享 |
| Cache | Cache 属性 | WB/WT/UC |

## TLB 组织

### 1. 全相联 TLB

```
┌─────────────────────────────────────────┐
│  全相联 TLB                             │
│  ┌─────────┬─────────┬─────────┐        │
│  │ Entry 0 │ Entry 1 │ Entry N │        │
│  └─────────┴─────────┴─────────┘        │
│         │            │           │      │
│         └────────────┼───────────┘      │
│                      │                  │
│               并行比较所有条目            │
└─────────────────────────────────────────┘
```

- **优点**：无冲突，命中率高
- **缺点**：硬件复杂，功耗高
- **适用**：小容量 TLB（32-64 条目）

### 2. 组相联 TLB

```
┌─────────────────────────────────────────┐
│  4 路组相联 TLB                          │
│  ┌────────────────────────────────────┐  │
│  │  Set 0: Way0  Way1  Way2  Way3    │  │
│  │  Set 1: Way0  Way1  Way2  Way3    │  │
│  │  Set 2: Way0  Way1  Way2  Way3    │  │
│  │  ...                              │  │
│  └────────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

- **优点**：平衡性能和复杂度
- **缺点**：可能有冲突
- **适用**：中等容量 TLB（64-256 条目）

### 3. 直接映射 TLB

```
┌─────────────────────────────────────────┐
│  直接映射 TLB                            │
│  ┌────────────────────────────────────┐  │
│  │  Index 0: Entry 0                 │  │
│  │  Index 1: Entry 1                 │  │
│  │  Index 2: Entry 2                 │  │
│  │  ...                              │  │
│  └────────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

- **优点**：硬件最简单
- **缺点**：冲突率高
- **适用**：大容量 TLB（256+ 条目）

### 组织对比

| 组织 | 面积 | 功耗 | 命中率 | 适用 |
|------|------|------|--------|------|
| 全相联 | 大 | 高 | 最高 | 小容量 |
| 组相联 | 中 | 中 | 高 | 中等容量 |
| 直接映射 | 小 | 低 | 中 | 大容量 |

## TLB 查找流程

### 基本流程

```
1. 输入虚拟地址 (VA)
2. 提取 VPN 和页内偏移
3. 用 VPN 索引 TLB
4. 比较 Tag（VPN + ASID）
5. 如果命中：
   - 检查 Valid 位
   - 检查权限（R/W/X/U）
   - 输出 PPN + 偏移 = PA
6. 如果未命中：
   - 触发 TLB 未命中异常
   - 或启动硬件页表遍历
```

### 流水化 TLB

```
Stage 1: VPN 索引 + Tag 比较
Stage 2: PPN 输出 + 权限检查
Stage 3: PA 生成
```

## TLB 替换策略

### LRU（Least Recently Used）

- 选择最久未使用的条目替换
- 命中率高
- 硬件复杂（需维护访问顺序）

### Pseudo-LRU

- 近似 LRU，硬件简单
- 命中率略低于 LRU
- 常用于组相联 TLB

### 随机替换

- 随机选择条目替换
- 硬件最简单
- 命中率较低

### FIFO（先进先出）

- 选择最早进入的条目替换
- 硬件简单
- 命中率介于 LRU 和随机之间

## TLB 刷新

### 刷新类型

| 类型 | 说明 | 指令 |
|------|------|------|
| 全刷新 | 刷新所有条目 | SFENCE.VMA（RISC-V） |
| ASID 刷新 | 刷新特定 ASID | SFENCE.VMA rs1, rs2 |
| 地址刷新 | 刷新特定虚拟地址 | SFENCE.VMA rs1 |
| VMID 刷新 | 刷新特定 VMID | HFENCE.VVMA |

### 刷新时机

- **上下文切换**：切换进程时刷新
- **页表修改**：修改页表后刷新
- **TLB 满**：需要分配新条目时替换
- **安全切换**：切换安全状态时刷新

### 刷新性能影响

- **全刷新**：性能影响大，需重新填充 TLB
- **局部刷新**：性能影响小，只刷新相关条目
- **ASID**：使用 ASID 避免上下文切换刷新

## 多级 TLB

### L1 TLB + L2 TLB

```
┌─────────────────────────────────────────┐
│  L1 TLB (小，快)                        │
│  - 全相联或 4 路组相联                   │
│  - 32-64 条目                           │
│  - 1 周期访问                           │
└──────────────────┬──────────────────────┘
                   │ 未命中
┌──────────────────▼──────────────────────┐
│  L2 TLB (大，慢)                        │
│  - 组相联                               │
│  - 256-1024 条目                        │
│  - 2-3 周期访问                         │
└─────────────────────────────────────────┘
```

### 优势

- **高命中率**：L1 未命中时 L2 可能命中
- **低延迟**：大多数访问在 L1 命中
- **可扩展**：L2 可以很大

---

## ASID（地址空间 ID）

### 为什么需要 ASID

- **避免刷新**：上下文切换时无需刷新 TLB
- **区分进程**：同一 VPN 不同进程映射不同 PA
- **提高性能**：减少 TLB 未命中

### ASID 位宽

| 架构 | ASID 位宽 | 地址空间数 |
|------|----------|-----------|
| RISC-V | 8-16 bit | 256-65536 |
| ARM | 8-16 bit | 256-65536 |
| x86 | 12 bit | 4096 |

### ASID 管理

- **静态分配**：每个进程固定 ASID
- **动态分配**：OS 动态分配 ASID
- **ASID 复用**：ASID 用完后复用

## 大页 TLB

### 大页 TLB 条目

| 页大小 | VPN 位数 | PPN 位数 | 覆盖范围 |
|--------|----------|----------|----------|
| 4KB | 27-36 bit | 27-36 bit | 4KB |
| 2MB | 18-27 bit | 18-27 bit | 2MB |
| 1GB | 9-18 bit | 9-18 bit | 1GB |

### 大页优势

- **减少 TLB 压力**：一个条目覆盖更大范围
- **提高命中率**：相同 TLB 大小覆盖更多内存
- **减少页表遍历**：更少的页表级数

## 设计注意事项

### RTL 实现要点

1. **Tag 比较**：需单周期完成
2. **权限检查**：在 TLB 查找时完成
3. **替换逻辑**：需维护替换状态
4. **刷新逻辑**：需支持多种刷新类型
5. **多级 TLB**：需管理 L1/L2 一致性

### 面积/功耗权衡

| 配置 | 面积 (KB) | 功耗 | 命中率 |
|------|----------|------|--------|
| 32 条目全相联 | 2-4 | 低 | ~90% |
| 64 条目 4 路 | 4-8 | 中 | ~95% |
| 256 条目 8 路 | 16-32 | 中高 | ~98% |
| 1024 条目 16 路 | 64-128 | 高 | ~99% |

### 常见陷阱

1. **TLB 刷新开销**：全刷新性能影响大
2. **ASID 管理**：ASID 有限，需复用
3. **大页对齐**：大页需自然对齐
4. **权限检查**：需在 TLB 查找时完成
5. **多级一致性**：L1/L2 TLB 需保持一致
