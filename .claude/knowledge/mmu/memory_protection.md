# 内存保护

> **用途**：处理器内存保护机制设计参考，供 CPU/SoC 安全子系统设计时检索
> **典型应用**：嵌入式、安全 SoC、虚拟化平台

---

## 概述

内存保护机制防止程序访问未授权的内存区域，确保系统安全和稳定。

### 保护类型

| 类型 | 说明 | 典型应用 |
|------|------|----------|
| 页保护 | 通过页表控制权限 | 所有支持虚拟内存的系统 |
| PMP | 物理内存保护 | 嵌入式、RISC-V |
| MPU | 内存保护单元 | ARM Cortex-M |
| IOMMU | I/O 内存保护 | 虚拟化、设备直通 |

---

## RISC-V PMP（物理内存保护）

### PMP 结构

```
┌─────────────────────────────────────────────┐
│  PMP (Physical Memory Protection)           │
│  ┌──────────────────────────────────────┐   │
│  │  PMP Entry 0: addr + cfg            │   │
│  │  PMP Entry 1: addr + cfg            │   │
│  │  ...                                │   │
│  │  PMP Entry 15: addr + cfg           │   │
│  └──────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

### PMP 寄存器

| 寄存器 | 说明 | 位宽 |
|--------|------|------|
| pmpcfg0-pmpcfg15 | 配置寄存器 | 64-bit (RV64) |
| pmpaddr0-pmpaddr15 | 地址寄存器 | 64-bit |

### PMP 配置字段

```
pmpcfg[i]:
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│  L  │  -  │  A  │  A  │  X  │  W  │  R  │     │
│     │     │ [1] │ [0] │     │     │     │     │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘

L: 锁定（M 模式可写，之后锁定）
A: 地址匹配模式
  0: 关闭
  1: TOR (Top of Range)
  2: NA4 (4-byte 自然对齐)
  3: NAPOT (Naturally Aligned Power-of-Two)
X: 可执行
W: 可写
R: 可读
```

### PMP 地址匹配

| 模式 | 说明 | 地址范围 |
|------|------|----------|
| TOR | 地址范围：pmpaddr[i-1] 到 pmpaddr[i] | 任意范围 |
| NA4 | 4 字节自然对齐 | 4 字节 |
| NAPOT | 2^n 字节自然对齐 | 2^n 字节 |

### PMP 优先级

- PMP 条目 0 优先级最高
- PMP 条目 15 优先级最低
- 匹配最高优先级条目

### PMP ePMP（增强 PMP）

新增配置：

| 字段 | 说明 |
|------|------|
| SRL | S 模式只读锁定 |
| SRW | S 模式读写权限 |
| SX | S 模式执行权限 |
| MRL | M 模式只读锁定 |
| MRW | M 模式读写权限 |
| MX | M 模式执行权限 |

## ARM MPU（内存保护单元）

### MPU 结构

```
┌─────────────────────────────────────────────┐
│  MPU (Memory Protection Unit)               │
│  ┌──────────────────────────────────────┐   │
│  │  Region 0: base + size + attr        │   │
│  │  Region 1: base + size + attr        │   │
│  │  ...                                │   │
│  │  Region 15: base + size + attr       │   │
│  └──────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

### MPU 寄存器

| 寄存器 | 说明 |
|--------|------|
| MPU_TYPE | MPU 类型 |
| MPU_CTRL | MPU 控制 |
| MPU_RNR | 区域号 |
| MPU_RBAR | 区域基地址 |
| MPU_RASR | 区域属性和大小 |

### MPU 区域属性

```
MPU_RASR:
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│  XN │  AP │ AP  │  TEX│  S  │  C  │  B  │     │
│     │ [2] │ [1:0]│    │     │     │     │     │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘

XN: 执行禁止
AP: 访问权限
  00: 无访问
  01: 特权读写
  10: 用户只读，特权读写
  11: 用户读写，特权读写
TEX/S/C/B: 内存类型
```

### MPU 区域大小

- 最小 32 字节
- 最大 4GB
- 必须是 2 的幂
- 必须自然对齐

## x86 保护机制

### 段保护

```
┌─────────────────────────────────────────────┐
│  段描述符                                    │
│  ┌─────────┬─────────┬─────────┬─────────┐  │
│  │  Base   │  Limit  │  Access │  Flags  │  │
│  └─────────┴─────────┴─────────┴─────────┘  │
└─────────────────────────────────────────────┘

Access 字段:
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│  P  │  DPL│  S  │  Type│     │     │     │     │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘

P: 存在位
DPL: 特权级 (0-3)
S: 系统段/用户段
Type: 段类型
```

### 页保护

```
页表项:
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│  NX │     │     │     │  U/S│  R/W│  P  │     │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘

NX: 执行禁止
U/S: 用户/特权
R/W: 读/写
P: 存在位
```

## 保护检查流程

### RISC-V PMP 检查

```
1. 获取物理地址 (PA)
2. 遍历 PMP 条目（0-15）
3. 检查每个条目的地址匹配
4. 如果匹配：
   - 检查权限（R/W/X）
   - 检查特权级（M/S/U）
   - 如果权限不足，触发异常
5. 如果无匹配：
   - M 模式：允许访问
   - S/U 模式：触发异常
```

### ARM MPU 检查

```
1. 获取物理地址 (PA)
2. 遍历 MPU 区域（0-15）
3. 检查每个区域的地址范围
4. 如果匹配：
   - 检查权限（AP）
   - 检查执行禁止（XN）
   - 如果权限不足，触发异常
5. 如果无匹配：
   - 默认内存类型
```

## 特权级保护

### RISC-V 特权级

| 特权级 | 名称 | 内存访问 |
|--------|------|----------|
| M | Machine | 完全访问 |
| S | Supervisor | 受 PMP 限制 |
| U | User | 受 PMP 限制 |

### ARM 特权级

| 特权级 | 名称 | 内存访问 |
|--------|------|----------|
| EL3 | Secure Monitor | 完全访问 |
| EL2 | Hypervisor | 受 MPU/页表限制 |
| EL1 | Kernel | 受 MPU/页表限制 |
| EL0 | User | 受 MPU/页表限制 |

## IOMMU（I/O 内存管理单元）

### IOMMU 功能

- **地址转换**：设备虚拟地址 → 物理地址
- **内存保护**：防止设备访问未授权内存
- **设备隔离**：隔离不同设备的内存访问
- **DMA 重映射**：重映射 DMA 地址

### IOMMU 结构

```
┌─────────────────────────────────────────────┐
│  IOMMU                                      │
│  ┌──────────────────────────────────────┐   │
│  │  页表遍历器                           │   │
│  │  (设备虚拟地址 → 物理地址)             │   │
│  └──────────────────────────────────────┘   │
│  ┌──────────────────────────────────────┐   │
│  │  TLB                                 │   │
│  │  (缓存地址转换)                       │   │
│  └──────────────────────────────────────┘   │
│  ┌──────────────────────────────────────┐   │
│  │  命令队列                             │   │
│  │  (IOMMU 命令)                        │   │
│  └──────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

### IOMMU 标准

| 标准 | 说明 | 典型应用 |
|------|------|----------|
| Intel VT-d | Intel IOMMU | x86 服务器 |
| ARM SMMU | ARM IOMMU | ARM SoC |
| RISC-V IOMMU | RISC-V IOMMU | RISC-V SoC |

## 安全扩展

### ARM TrustZone

```
┌─────────────────────────────────────────────┐
│              Normal World                    │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐    │
│  │  EL0    │  │  EL1    │  │  EL2    │    │
│  │ (App)   │  │ (Linux) │  │ (KVM)   │    │
│  └─────────┘  └─────────┘  └─────────┘    │
├─────────────────────────────────────────────┤
│              Secure World                    │
│  ┌─────────┐  ┌─────────┐                  │
│  │  EL0    │  │  EL1    │                  │
│  │(Trusted)│  │(OP-TEE) │                  │
│  └─────────┘  └─────────┘                  │
├─────────────────────────────────────────────┤
│              EL3 (Secure Monitor)            │
│         ARM Trusted Firmware (ATF)          │
└─────────────────────────────────────────────┘
```

- **安全世界**：可访问所有内存
- **普通世界**：只能访问非安全内存
- **EL3**：控制安全状态切换

### RISC-V 安全扩展

- **PMP**：物理内存保护
- **ePMP**：增强 PMP，支持 S 模式权限
- **安全区域**：通过 PMP 实现安全隔离

## 设计注意事项

### RTL 实现要点

1. **检查逻辑**：需在内存访问时检查权限
2. **优先级**：需处理多个保护区域重叠
3. **性能**：保护检查不应增加关键路径延迟
4. **异常处理**：需处理权限故障异常
5. **虚拟化**：需支持两阶段保护

### 面积/功耗权衡

| 配置 | 面积 | 功耗 | 说明 |
|------|------|------|------|
| PMP 8 条目 | 小 | 低 | 嵌入式 |
| PMP 16 条目 | 中 | 中 | 通用 |
| MPU 8 区域 | 小 | 低 | ARM Cortex-M |
| MPU 16 区域 | 中 | 中 | ARM Cortex-R |

### 常见陷阱

1. **检查延迟**：保护检查不应增加关键路径
2. **优先级冲突**：多个保护区域重叠时需仲裁
3. **虚拟化开销**：两阶段保护增加延迟
4. **安全隔离**：需确保安全世界和普通世界隔离
5. **调试访问**：调试时可能需要绕过保护
