# FS↔微架构 数据交换协议（共享）

> FS（功能规格）与微架构文档之间的章节映射、版本同步和评审协作规则。
> 供 chip-fs-writer 和 chip-microarch-writer 共同引用，不得在 agent 定义中内联复制。

---

## 1. FS→微架构章节映射

| FS 章节 | 微架构章节 | 继承方式 |
|---------|-----------|----------|
| §3 概述 | §3 子模块概述 | 继承定位和功能描述 |
| §5 子模块设计 | §4 接口 + §5 微架构 | 端口继承，内部实现展开 |
| §6 顶层接口 | §4.1 端口列表 | 信号直接继承 |
| §7 寄存器定义 | §5.6 IP 集成（如涉及寄存器访问） | 检查是否影响寄存器访问逻辑 |
| §8 PPA | §8 PPA 预估 | 预算作为约束上限 |
| §9 时钟复位 | §7 时钟与复位 | 策略继承 |
| §14 RTM | §13 RTM | 状态继承并细化 |

## 2. 版本同步规则

**触发条件**：FS 版本号变更时，下游微架构文档必须同步检查。

| FS 变更类型 | 微架构影响章节 | 同步要求 |
|------------|---------------|----------|
| §5 端口列表变更 | §4 接口定义 | 必须同步 |
| §6 顶层接口变更 | §4.1 端口列表 | 必须同步 |
| §7 寄存器变更 | §5.6 IP 集成 | 检查影响 |
| §8 PPA 变更 | §8 PPA 预估 | 必须同步 |
| §9 时钟复位变更 | §7 时钟与复位 | 必须同步 |
| §4 功能变更 | §5 微架构设计 | 检查影响 |

**大版本变更（v1.x→v2.0）**：按上表逐行评估影响范围，仅受影响章节需要重写，非变更章节继承原版本。

**版本号规则**：
- FS 小版本变更（v1.0→v1.1）：微架构对应小版本递增
- FS 大版本变更（v1.x→v2.0）：微架构逐章评估影响，仅受影响章节重写

**下游通知**：FS 变更后，微架构文档作者需在微架构 §2 修订历史中记录 FS 版本变更引用。

## 3. 评审协作

FS 完成后，Reviewer（chip-arch-reviewer）需要的输入：

| 输入 | 用途 | 对应 QC |
|------|------|---------|
| 主文档 `.md` | 全文评审 | — |
| REQ 汇总表 | 核对数值一致性 | QC-14 |
| D2 源文件 | 验证图表可编译 | QC-18 |
| 交付物清单 | 检查完整性 | QC-08 |

微架构完成后，Reviewer 需要的额外输入：
- FS 文档（核对继承一致性，对应映射表 §1）
- 微架构 §13 RTM（与 FS §14 RTM 比对）
