# FS↔微架构 数据交换协议（共享）

> FS（功能规格）与微架构文档之间的章节映射、版本同步和评审协作规则。
> 供 chip-fs-writer 和 chip-microarch-writer 共同引用，不得在 agent 定义中内联复制。

---

## 0. 模块拆分约束（铁律）

### 0.1 顶层模块：纯连接，零逻辑

**规则**：顶层模块（`{module}_top`）的微架构文档中：
- **§5 微架构设计** 仅包含子模块实例化列表和信号连接拓扑，**禁止**包含任何组合逻辑（`always @(*)`）或时序逻辑（`always @(posedge clk)`）
- **§3.3 内部框图** 展示子模块间互联关系，不展示顶层内部逻辑
- **§4 接口定义** 仅定义顶层对外端口到子模块端口的映射关系
- 若 FS §4/§5 分析发现需要额外功能模块（如仲裁器、协议转换、数据格式化等），**必须**拆分为独立子模块，为其生成完整微架构文档

**判定标准**：顶层模块的 RTL 实现中不应出现 `always` 块（除连线外），仅包含 `assign` 语句和子模块实例化。

### 0.2 寄存器模块：纯配置读写，功能表项归位

**规则**：寄存器模块（`{module}_reg_mod`）的微架构文档中：
- **仅包含**：APB 从接口、寄存器地址解码、配置寄存器读写逻辑、W1C/W0C 中断状态寄存器
- **不包含**：任何功能逻辑（仲裁、数据处理、状态机、FIFO 管理等）
- FS §7 中定义的寄存器按**功能相近原则**分配：
  - 纯配置寄存器（CTRL、INT_MASK、CH_EN 等）→ 寄存器模块
  - 功能关联寄存器（CREDIT_CFG → 信用管理模块、ENCRYPT_CFG → 加密模块）→ 对应功能模块
  - 状态只读寄存器（STATUS）→ 寄存器模块提供读接口，实际状态值由功能模块产生
- 寄存器模块对外提供**寄存器读写接口**（`reg_wr_en`/`reg_rd_en`/`reg_addr`/`reg_wr_data`/`reg_rd_data`），功能模块自行采样配置值

**判定标准**：寄存器模块的 RTL 实现中不应出现数据通路逻辑或功能状态机，仅包含地址解码和寄存器读写。

## 1. FS→微架构章节映射

| FS 章节 | 微架构章节 | 继承方式 |
|---------|-----------|----------|
| §3 概述 | §3 子模块概述 | 继承定位和功能描述 |
| §5 子模块设计 | §4 接口 + §5 微架构 | 端口继承，内部实现展开 |
| §6 顶层接口 | §4.1 端口列表 | 信号直接继承 |
| §7 寄存器定义 | §5.6 寄存器分配表 | 按功能归属原则分配：纯配置→寄存器模块，功能关联→功能模块 |
| §8 PPA | §8 PPA 预估 | 预算作为约束上限 |
| §9 时钟复位 | §7 时钟与复位 | 策略继承 |
| §14 RTM | §13 RTM | 状态继承并细化 |

## 2. 版本同步规则

**触发条件**：FS 版本号变更时，下游微架构文档必须同步检查。

| FS 变更类型 | 微架构影响章节 | 同步要求 |
|------------|---------------|----------|
| §5 端口列表变更 | §4 接口定义 | 必须同步 |
| §6 顶层接口变更 | §4.1 端口列表 | 必须同步 |
| §7 寄存器变更 | §5.6 寄存器分配表 + 寄存器模块/功能模块接口 | 检查影响（需重新评估寄存器归属） |
| §8 PPA 变更 | §8 PPA 预估 | 必须同步 |
| §9 时钟复位变更 | §7 时钟与复位 | 必须同步 |
| §4 功能变更 | §5 微架构设计 | 检查影响 |

**大版本变更（v1.x→v2.0）**：按上表逐行评估影响范围，仅受影响章节需要重写，非变更章节继承原版本。

**版本号规则**：
- FS 小版本变更（v1.0→v1.1）：微架构对应小版本递增
- FS 大版本变更（v1.x→v2.0）：微架构逐章评估影响，仅受影响章节重写

**下游通知**：FS 变更后，微架构文档作者需在微架构 §2 修订历史中记录 FS 版本变更引用。

## 3. 评审协作

FS 完成后，Reviewer（chip-arch-reviewer）需要的输入：

| 输入 | 用途 | 对应 QC |
|------|------|---------|
| 主文档 `.md` | 全文评审 | — |
| REQ 汇总表 | 核对数值一致性 | QC-14 |
| D2 源文件 | 验证图表可编译 | QC-18 |
| 交付物清单 | 检查完整性 | QC-08 |

微架构完成后，Reviewer 需要的额外输入：
- FS 文档（核对继承一致性，对应映射表 §1）
- 微架构 §13 RTM（与 FS §14 RTM 比对）
