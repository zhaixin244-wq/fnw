# 降级策略（共享）

> 外部资源不可用时的处理规则。**禁止因资源缺失而中断工作流。**

| 场景 | 降级行为 |
|------|----------|
| RAG 检索无结果 | 标注 **"基于通用知识"**，利用自身经验继续，关键结论旁注明来源缺失 |
| RAG 检索返回矛盾信息 | 标注两个来源差异，以最新版本为准；无法判断时暂停请用户确认 |
| 输入不完整 | **暂停**，列出所有缺失输入项，等待用户补充后继续 |
| Skills 调用失败 | 将该功能**内化执行**，输出中注明 "内化执行" |
| D2 编译失败 | 保留 `.d2` 源文件，标注 `[D2-DEGRADED]`，降级为文本描述 |
| Wavedrom JSON 格式错误 | 保留 `.json` 源文件，降级为文本时序表，标注错误位置 |
| 图表批量编译部分失败 | 成功项正常输出，失败项标注降级，不阻断整体流程 |

**输入优先级**：各 agent 定义中自行指定 Must/Should/Could 分级及缺失时行为。本文件仅定义通用降级三原则（标注来源→暂停等确认→内化执行），不重复定义具体输入的优先级。

**适用范围**：所有芯片架构 Agent。
