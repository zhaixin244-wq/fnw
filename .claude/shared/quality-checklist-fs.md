## 执行顺序

**两阶段自检**：先执行脚本可查项（快速过滤硬性错误），再执行人工检查项。

| 阶段 | 范围 | 耗时参考 | 通过条件 |
|------|------|----------|----------|
| 第一阶段（自动化） | QC-02/05/08/09/11/12/17/18/20/21 | ~2 min | 10 项全部通过 |
| 第二阶段（人工） | QC-01/03/04/06/07/10/13/14/15/16/19/22 | ~10 min | 12 项全部通过 |

第一阶段任一失败 → 修复后重新执行第一阶段，通过后再进入第二阶段。

## 自动化检查指引

可脚本检查的 QC 项（标注"脚本可查"）用以下方式验证：

| QC | 检查方法 |
|----|----------|
| QC-02 | 提取 §7.1 偏移地址列，检查重复值 |
| QC-05 | 统计 §14 REQ 行数 vs 上游 REQ 总数，计算覆盖率 |
| QC-08 | `ls /ds/doc/fs/` + `ls /ds/doc/fs/tmp/` 验证文件存在 |
| QC-09 | `d2 --layout <file.d2>` 编译验证 + Wavedrom JSON 格式校验 |
| QC-11 | 提取 §8.4 合计行与 §8.1~8.3 数值比较 |
| QC-12 | 提取 §7.2 每个寄存器 bit 范围，检查 0~(MSB) 全覆盖 |
| QC-17 | 提取 §5.x 编号序列，检查连续性 |
| QC-18 | `find /ds/doc/fs/tmp/ -name "*.d2" -exec d2 --layout dagre {} \;` |
| QC-20 | grep §5.x.2 中 `\[\d+:\d+\]` 硬编码模式 |
| QC-21 | 提取 §3.1 F-xxx 与 §4.1 REQ-xxx，交叉比对 |
