# RTL 实现质量自检清单（共享）

> RTL 代码实现完成后必须执行。从微架构 QC 继承设计意图验证项，新增 RTL 特有检查项（IC-01~39 + IM-01~08）。
> 每个 agent 调用时引用本文件，不得内联复制。

---

## ⚠️ 强制质量门禁声明

| 门禁类型 | 强制级别 | 通过标准 | 失败行为 | 跳过权限 |
|----------|----------|----------|----------|----------|
| **Lint 检查** | **MUST** | `lint_summary.log` 输出 ALL PASS | 必须进入自愈循环修复，禁止交付 | **禁止跳过** |
| **综合检查** | **MUST** | `synth_summary.log` 输出 ALL PASS + 面积差异 <50% | 必须进入自愈循环优化，禁止交付 | **禁止跳过** |
| **自检** | **MUST** | IC-01~39 + IM-01~08 全部通过 | 必须逐项修复后重新自检 | **禁止跳过** |

**违反门禁的交付物一律视为无效，chip-arch-reviewer 有权拒绝评审。**

## 矛盾检测规则（IM-01~08）

> 检查 RTL 实现与微架构文档间的矛盾。

| # | 规则 | 触发条件 | 行为 |
|---|------|----------|------|
| IM-01 | 端口位宽不一致 | RTL 端口位宽 ≠ 微架构 §4.1 | 标记 Critical，暂停确认 |
| IM-02 | FSM 状态缺失 | 微架构 §5.3 定义的状态在 RTL 中不存在 | 标记 Critical |
| IM-03 | FIFO 深度不一致 | RTL FIFO 参数 ≠ 微架构 §5.5 | 标记 Critical |
| IM-04 | CBB 实例化参数不一致 | RTL CBB 参数 ≠ 微架构 §5.6 | 标记 Major |
| IM-05 | 背压链路断裂 | 微架构 §5.2 定义的反压路径在 RTL 中缺失 | 标记 Critical |
| IM-06 | 缺少微架构规定的 SVA | 微架构 §10 定义的验证场景在 SVA 中未实现 | 标记 Minor |
| IM-07 | SDC 时钟周期不一致 | RTL SDC 时钟周期 ≠ 微架构 §6 | 标记 Major |
| IM-08 | CBB 缺失未标记 | 使用了 CBB 功能但无 `// CBB Ref` 且无 `[CBB-MISSING]` | 标记 Major |

## 执行顺序

**三阶段自检**：

| 阶段 | 范围 | 耗时参考 | 通过条件 |
|------|------|----------|----------|
| 第一阶段（自动化） | IC-06/07/13/22/34/35/36 | ~2 min | 7 项全部通过 |
| 第二阶段（编码规范） | IC-01~05/08~12/14~21 | ~5 min | 18 项全部通过 |
| 第三阶段（设计模式+DFT+组织） | IC-23~33/37~39 + 矛盾检测 | ~10 min | 17+8 项全部通过 |

第一阶段任一失败 → 修复后重新执行第一阶段，通过后再进入第二阶段。

**Lint 门禁**：Verilator `--lint-only -Wall` 无 Error 级别告警。

**综合门禁**：yosys 综合 ALL PASS + 面积差异 <50%。

> **⚠️ 强制门禁执行顺序**：
> 1. **Lint 检查**（MUST）→ ALL PASS 才能进入下一步
> 2. **综合检查**（MUST）→ ALL PASS + 面积达标 才能进入下一步
> 3. **自检**（MUST）→ IC-01~39 + IM-01~08 全部通过才允许交付
>
> **禁止跳过任何门禁，禁止降级门禁标准。**

## 自动化检查指引

| IC | 检查方法 |
|----|----------|
| IC-01 | grep RTL 中 `always @(posedge clk` 块，检查 reset 分支是否覆盖所有 `<=` 左侧信号 |
| IC-04 | grep RTL 中 `case` 关键字，检查每个 case 块是否包含 `default` |
| IC-07 | grep RTL 中常量模式 `[0-9]+'[dhb][0-9a-fA-F]+`，检查是否指定位宽 |
| IC-10 | grep RTL 中 `\[\d+:\d+\]` 硬编码模式 |
| IC-11 | grep RTL 中 `begin : ` 是否在 generate 块内 |
| IC-13 | grep RTL 中 `casex\|casez` |
| IC-17 | grep RTL 中 `if (rst_n` 或 `if(rst_n` 检查极性 |
| IC-18 | grep RTL 中 FIFO 实例化参数，验证深度 = 2^n |
| IC-22 | grep RTL 中子模块实例化是否使用 `.` 开头的名称关联 |
| IC-34 | grep RTL 中 ICG 实例化是否包含 `.TE(scan_en)` 或 `.scan_en(` 端口 |
| IC-35 | 统计非空行中 `//` 和 `/*` 出现行数占比 |
| IC-36 | 统计每个 always 块的行数 |
