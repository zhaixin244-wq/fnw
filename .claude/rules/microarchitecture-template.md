## 9. RTL 实现指导

### 文件结构
```
{submodule_name}.v          — 主模块 RTL
{submodule_name}_intf.sv    — Interface（如需）
{submodule_name}_sva.sv     — SVA 断言
{submodule_name}_tb.v       — 仿真 testbench
```
划分原则：FSM>10状态 → 独立文件，FIFO → 独立文件，Interface信号>15 → 独立文件。

### 参数化
派生参数用 `localparam`，端口位宽用参数表示，禁止硬编码 `[31:0]`。

### 编码规范
遵循 `rules/coding-style.md`，本子模块特别注意：

| 规则 | 优先级 |
|------|--------|
| 时序逻辑仅用 `always @(posedge clk or negedge rst_n)` | Must |
| 组合逻辑 `always @(*)` 必须赋默认值 | Must |
| case 必须有 default | Must |
| 禁止门控时钟，使用标准 ICG | Must |
| 参数化位宽 | Must |
| FSM 用 localparam，禁止 `define | Must |
| 同一 always 块禁止混用 `<=` 和 `=` | Must |
| generate 块必须有标签 | Must |

### 可综合性检查点

| 检查项 | 状态 |
|--------|------|
| 所有寄存器有明确复位值 | □ |
| 组合逻辑无 latch | □ |
| 时序 `<=` / 组合 `=` / 不混用 | □ |
| case 有 default | □ |
| 无门控时钟 | □ |
| 无不可综合结构 | □ |
| 位宽匹配 | □ |
| SVA 在 ifdef 内 | □ |
| 参数化位宽 | □ |
| generate 有标签 | □ |

### 反合理化清单

| 偷懒借口 | 架构师的回应 |
|----------|------------|
| "信号名不重要" | 信号名是接口契约 |
| "默认值设 0 就行" | 必须符合协议规范 |
| "加个 always 就够了" | 组合/时序由数据通路决定 |
| "复位异步无所谓" | 影响 CDC、DFT、面积 |
| "先写 RTL 再补断言" | 断言是可执行文档 |
| "时序后面综合再看" | 关键路径必须在架构阶段分析 |
| "FIFO 深度随便选" | 深度 = 流控模型计算结果 |
| "if-else 不用写 else" | 不写 else = latch |
| "位宽差不多就行" | 仿真可能通过但硅片失败 |
| "interface 里加个 always" | Interface 不含逻辑 |

### SVA 断言要点
- 握手稳定性：`valid && !ready |=> valid`
- 数据稳定性：`valid && !ready |=> $stable(data)`
- 非法状态检测：`state_cur inside {合法状态集}`
- 用 `bind` 绑定到 RTL，放在 `` `ifdef ASSERT_ON `` 内

### 覆盖率模型

| 覆盖组 | 覆盖点 | 目标 |
|--------|--------|------|
| FSM 状态/转移 | 所有状态和转移 | 100% |
| 握手组合 | valid×ready 4种 | 100% |
| 数据边界值 | 0, MAX, MAX-1, 1 | 100% |
| 异常场景 | 每种异常 | 100% |
