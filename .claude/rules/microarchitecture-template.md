## 特殊子模块编写指引

> **顶层模块（`_top`）和寄存器模块（`_reg_mod`）有专用编写规则，详见下方。其他子模块按标准模板编写。**

### 顶层模块（`{module}_top`）编写规则

**铁律：顶层模块仅做子模块实例化和信号连接，禁止包含任何逻辑。**

| 章节 | 顶层模块特殊处理 |
|------|-----------------|
| §3 概述 | 描述模块整体功能和子模块组成，不描述顶层自身逻辑 |
| §3.3 内部框图 | **必须生成模块连接图**（使用 `chip-png-d2-gen`），展示子模块间互联拓扑、外部接口和内部接口，不展示顶层内部逻辑 |
| §4 接口定义 | 仅定义顶层对外端口到子模块端口的映射表 |
| §5 微架构 | **仅 §5.6 子模块实例化列表**（含参数和端口连接），§5.1~5.5 全部标注"不适用——顶层无逻辑" |
| §6 时序 | 标注"时序由各子模块独立约束，顶层无关键路径" |
| §8 PPA | 汇总各子模块 PPA，顶层自身面积 ≈ 0（仅连线） |
| §9 RTL 指导 | 强调：仅 `assign` + 子模块实例化，禁止 `always` 块 |

**模块连接图要求**：
- 文件命名：`wd_{module}_top_connect.d2`
- 内容：所有子模块作为节点，外部接口作为边界节点，连线标注接口名称
- 配色：子模块/蓝色系 `#EBF5FB`，外部输入/绿色系 `#E8F8F5`，外部输出/红色系 `#FDEDEC`
- 形状：子模块 → 矩形，外部接口 → 六边形

```markdown
![{模块名} 模块连接图](wd_{module}_top_connect.png)
```

**FS 分析发现需要额外模块时**：暂停顶层编写，先为新发现的子模块生成完整微架构文档，再继续顶层集成。

### 寄存器模块（`{module}_reg_mod`）编写规则

**铁律：寄存器模块仅包含配置寄存器读写逻辑，不承载任何功能逻辑。**

| 章节 | 寄存器模块特殊处理 |
|------|-------------------|
| §3 概述 | 描述寄存器模块职责：APB 从接口 + 地址解码 + 配置寄存器读写 |
| §4 接口定义 | APB 从接口（§6 对应信号）+ 寄存器读写接口（`reg_wr_en`/`reg_rd_en`/`reg_addr`/`reg_wr_data`/`reg_rd_data`） |
| §5 微架构 | §5.1 数据通路 = APB 写路径 + APB 读路径；§5.2 控制 = 地址解码逻辑；§5.3 状态机 = 无（纯组合+寄存器） |
| §5.6 寄存器分配表 | **关键**：列出 FS §7 所有寄存器，标注每个寄存器的归属模块（本模块 / 功能模块） |
| §8 PPA | 仅含 APB 接口 + 地址解码 + 寄存器阵列面积 |

**寄存器归属判定规则**：

| 寄存器类型 | 归属 | 示例 |
|-----------|------|------|
| 纯控制/使能寄存器 | 寄存器模块 | CTRL、INT_MASK、CH_EN |
| 中断状态（W1C） | 寄存器模块 | INT_STATUS |
| 功能配置表项 | **功能模块** | CREDIT_CFG → 信用管理模块、ENCRYPT_CFG → 加密模块 |
| 状态只读寄存器 | 寄存器模块（读接口）+ 功能模块（值产生） | STATUS |
| FIFO 阈值/深度配置 | **功能模块** | FIFO_THR → 对应 FIFO 所属模块 |

**寄存器读写接口定义**：

```verilog
// 寄存器模块 → 功能模块（单向广播）
output reg                   reg_wr_en,      // 寄存器写使能
output reg                   reg_rd_en,      // 寄存器读使能
output reg  [ADDR_WIDTH-1:0] reg_addr,       // 寄存器地址
output reg  [DATA_WIDTH-1:0] reg_wr_data,    // 写数据
input  wire [DATA_WIDTH-1:0] reg_rd_data_xx, // 各功能模块返回的读数据（多路 mux）
```

## 9. RTL 实现指导

### 文件结构
```
{submodule_name}.v          — 主模块 RTL
{submodule_name}_intf.sv    — Interface（如需）
{submodule_name}_sva.sv     — SVA 断言
{submodule_name}_tb.v       — 仿真 testbench
```
划分原则：FSM>10状态 → 独立文件，FIFO → 独立文件，Interface信号>15 → 独立文件。

### 参数化
派生参数用 `localparam`，端口位宽用参数表示，禁止硬编码 `[31:0]`。

### 编码规范

遵循 `rules/coding-style.md`。本子模块在 RTL 实现阶段需特别关注以下高风险项：

| 高风险项 | 关注点 |
|----------|--------|
| 组合逻辑 latch 防护 | always @(*) 输出默认值、if 补 else、case 有 default |
| FSM 非法状态回收 | 所有非法状态必须回到 IDLE |
| FIFO 深度计算 | 必须基于流控模型，深度为 2 的幂 |
| CDC 处理 | 单 bit 双触发器，多 bit 异步 FIFO/格雷码 |

> **RTL 实现检查清单**（IC-01~39 + IM-01~08）统一维护在 `shared/quality-checklist-impl.md`，本模板不重复列出。

### 反合理化清单

> 以下为本子模块设计过程中需警惕的常见误区。完整清单见 `rules/coding-style.md` §16。

| 偷懒借口 | 架构师的回应 |
|----------|------------|
| "时序后面综合再看" | 关键路径必须在架构阶段分析 |
| "FIFO 深度随便选" | 深度 = 流控模型计算结果 |
| "复位异步无所谓" | 影响 CDC、DFT、面积 |
| "先写 RTL 再补断言" | 断言是可执行文档 |
