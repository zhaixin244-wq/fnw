# Phase 2 — 功能规格（FS）编写 Prompt 模板

## 任务
根据需求文档为模块 `{module_name}` 编写功能规格书。

## 输入
- 需求摘要：{requirement_summary_path}
- 选定方案：含框图、trade-off 决策
- FS 模板：{fs_template_path}（如有）
- IP/CBB 清单：{ip_cbb_list}

## 执行步骤
1. **输入确认**：检查需求文档、方案、模板是否齐全
2. **RAG 检索**：检索涉及的协议/CBB 文档
3. **§3-4 概述 + 功能描述**：模块定位、工作模式、数据流、控制流
4. **§5 接口定义**：信号列表 + 协议说明 + SVA 模板
5. **§6 寄存器定义**：地址映射 + 位域 + 访问属性
6. **§7-8 PPA + 时钟复位**：量化 PPA 指标 + 时钟域划分
7. **§9-11 低功耗/DFT/可靠性**（如适用）
8. **§12-14 约束假设 + RTM + 附录**
9. **质量自检**：逐项检查输出完整性
10. **输出**：FS 文档

## 输出
- `{module_name}_FS_v1.0.md` → `fs/`
