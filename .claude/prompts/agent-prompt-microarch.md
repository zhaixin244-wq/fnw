# Phase 3 — 微架构文档编写 Prompt 模板

## 任务
根据 FS 文档为模块 `{module_name}` 的子模块 `{submodule_name}` 编写微架构规格书。

## 输入
- FS 文档：{fs_doc_path}
- 微架构模板：{microarch_template_path}（如有）
- IP/CBB 文档：{ip_cbb_docs}
- 关键约束：频率={freq}, 面积={area}, 功耗={power}

## 执行步骤
1. **输入确认**：检查 FS、模板、IP/CBB 文档是否齐全
2. **RAG 检索**：检索涉及的协议/CBB/编码规范文档
3. **概述 + 框图**：功能定位、与兄弟模块关系、子模块框图
4. **接口定义**：端口列表 + 协议时序 + FS 接口映射
5. **数据通路**：完整数据流图 + 关键路径 + 时序裕量
6. **控制逻辑 + 状态机**：控制信号 + FSM 设计
7. **流水线/FIFO/IP 集成**（如适用）：FIFO 深度计算、IP 实例化
8. **时序分析 + 时钟复位 + CDC**：关键路径识别 + 复位策略
9. **PPA 预估**：逻辑面积 + 存储面积 + 功耗 + 延迟
10. **RTL 指导 + 反合理化清单 + SVA 模板**
11. **质量自检**

## 输出
- `{module_name}_{submodule_name}_microarch_v1.0.md` → `microarch/`

## 并行规则
- 子模块间无依赖时可并行编写
- 顶层微架构在所有子模块完成后编写
- 接口以 FS 定义为准保持一致
