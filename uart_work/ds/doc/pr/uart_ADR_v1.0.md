# UART 架构决策记录

## ADR-001: 波特率生成方案选择

**状态**：accepted

**背景**：需要在 50MHz 系统时钟下生成 9600~921600 bps 的波特率时钟。整数分频在某些波特率下误差较大（如 50MHz/16/115200 = 27.127，整数分频为 27，误差 0.47%）。

**决策**：采用小数分频方案（16 位整数 + 4 位小数累加器）。

**替代方案**：
- 方案 A（整数分频）：实现简单，但精度受限
- 方案 B（小数分频）：精度高，面积略增

**后果**：
- 正面：波特率误差 < 0.1%，满足所有标准波特率
- 负面：增加约 20 个寄存器（4 位累加器 + 16 位计数器）
- 风险：无

## ADR-002: FIFO 实现方式

**状态**：accepted

**背景**：TX/RX FIFO 需要 16 字节深度。

**决策**：使用寄存器阵列实现。

**替代方案**：
- 方案 A（寄存器阵列）：深度 ≤ 16 时面积可接受，时序好
- 方案 B（SRAM）：深度 > 16 时面积优势明显

**后果**：
- 正面：无需 SRAM IP，时序简单，综合工具自动优化
- 负面：深度扩展到 32+ 时面积增大
- 风险：无

## ADR-003: 过采样倍数

**状态**：accepted

**背景**：UART 接收需要在数据位中间采样，过采样倍数影响精度。

**决策**：默认 16x 过采样，可配置为 8x。

**后果**：
- 正面：16x 模式下精度高，抗噪声能力强
- 负面：16x 模式下需要更高的 baud_tick 频率
- 风险：无

---

## ADR-004: 子模块 FIFO 独立文件

**状态**：accepted

**背景**：FIFO 是通用 CBB，可能被多个模块复用。

**决策**：`uart_fifo` 作为独立子模块文件，参数化设计。

**后果**：
- 正面：代码复用性好，可独立验证
- 负面：无
- 风险：无

---

## ADR-005: 寄存器接口兼容 16550

**状态**：accepted

**背景**：16550 是业界标准 UART 控制器，软件驱动生态成熟。

**决策**：寄存器布局兼容 16550，扩展寄存器放在高位地址。

**后果**：
- 正面：可复用现有 16550 驱动
- 负面：部分 16550 特性不需要（如 MODEM 控制简化）
- 风险：无
