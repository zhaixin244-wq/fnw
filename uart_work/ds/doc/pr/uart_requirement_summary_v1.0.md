## 1. 模块定位

UART（Universal Asynchronous Receiver/Transmitter）模块作为 SoC 的低速外设通信接口，负责串行异步数据的发送与接收。通过 APB 总线接口连接系统总线，为软件提供配置和数据通路。

### 1.1 应用场景

| 场景 | 说明 |
|------|------|
| 调试接口 | SoC 调试日志输出至 PC 串口终端 |
| 低速外设通信 | 连接 GPS 模块、蓝牙模块、传感器等 |
| 固件升级 | 通过串口进行 bootloader 数据传输 |
| 系统控制 | AT 指令交互、CLI 命令行接口 |

### 1.2 在 SoC 中的位置

```
CPU ─── AXI/AHB ─── APB Bridge ─── UART ─── 串行线 ─── 外部设备
                         │
                    ┌────┴────┐
                    │ APB Bus │
                    └────┬────┘
                         │
                    ┌────┴────┐
                    │  UART   │
                    │ (本模块) │
                    └────┬────┘
                         │
                    txd / rxd / rts_n / cts_n
```

## 3. 方案比选

### 3.1 波特率生成方案

| 方案 | 描述 | 优点 | 缺点 | 选择 |
|------|------|------|------|------|
| A. 固定分频 | 系统时钟 / (16 × 波特率) - 1 | 简单、面积小 | 精度受限于整数分频 | 用于低波特率 |
| B. 小数分频 | 整数分频 + 小数累加器 | 精度高、支持任意波特率 | 稍复杂 | **采用** |

**采用方案 B**：小数分频（Fractional Baud Rate Generator），通过 16 位整数分频寄存器 + 4 位小数累加器实现高精度波特率生成。

### 3.2 FIFO 实现方案

| 方案 | 描述 | 优点 | 缺点 | 选择 |
|------|------|------|------|------|
| A. 寄存器阵列 | 用寄存器实现 FIFO | 时序好、简单 | 面积大 | 深度 ≤ 8 时使用 |
| B. SRAM | 用 SRAM 实现 FIFO | 面积小 | 需要 SRAM IP | 深度 > 8 时使用 |

**采用方案 B**：TX/RX FIFO 深度 16 字节，使用寄存器阵列实现（深度 = 16，处于边界，优先时序简单性）。

### 3.3 寄存器接口方案

| 方案 | 描述 | 选择 |
|------|------|------|
| A. APB | 标准 APB 从接口 | **采用** |
| B. AHB | AHB-Lite 从接口 | 不采用（UART 低速，APB 足够） |

## 4. PPA 目标

| 指标 | 目标值 | 单位 |
|------|--------|------|
| 频率 | ≥ 50 | MHz |
| 动态功耗 | < 5 | mW |
| 静态功耗 | < 0.5 | mW |
| 逻辑面积 | < 10 | kGates |
| TX FIFO | 16 × 10 bit | bit |
| RX FIFO | 16 × 11 bit | bit |

| 约束项 | 值 | 说明 |
|--------|-----|------|

| 波特率范围 | 9600 ~ 921600 bps | 覆盖常见应用场景 |

## 6. 风险评估

| ID | 风险 | 概率 | 影响 | 缓解方案 |
|----|------|------|------|----------|
| R-001 | 波特率精度不足 | L | M | 小数分频方案，4 位小数精度 |
| R-002 | FIFO 溢出导致数据丢失 | M | H | 溢出标志 + 中断 + 流控 |
| R-003 | RX 亚稳态 | L | H | 2 级同步器 + 过采样 |
