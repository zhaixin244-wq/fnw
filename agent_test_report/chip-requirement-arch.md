## 2. 测试场景与结果

### 场景 1：完整需求输入

**输入描述**：AXI4-Lite CRC 校验模块，28nm 工艺，200MHz，64bit 数据位宽，延迟 ≤5cycles，面积 ≤30kGates，功耗 ≤10mW，含完整约束维度（≥6项）

**预期路径**：`complete` → stage0 → stageA → stageB 核对 → stageC0 → stageC

**执行过程**：

| 步骤 | 执行内容 | 结果 |
|------|----------|------|
| 代办清单 | 输出待办事项清单 | ✅ |
| stage0 | 探索模块定位、核心功能、关键约束、初步架构方向 | ✅ |
| stageA | 4 问按序确认（位置/功能/PPA优先级/探索结论） | ✅ |
| stageB | 9 项约束逐项确认，已有项直接 ✅，缺失项补问 | ✅ 全部确认 |
| stageC0 | 矛盾检测 CD-001~CD-005 | ✅ CD-004 边界风险检测到 |
| stageC | 需求确认汇总表 | ✅ 完整格式 |

**矛盾检测详情**：

| 矛盾编号 | 检测结果 | 说明 |
|----------|----------|------|
| CD-001（频率 vs 工艺） | 通过 | 28nm @ 200MHz 在典型 Fmax 400-500MHz 范围内 |
| CD-002（延迟 vs 频率） | 通过 | ≤5cycles 合理 |
| CD-003（面积 vs 功能） | 通过 | AXI-Lite + CRC ≤30kGates 合理 |
| CD-004（功耗 vs 工艺/频率） | ⚠️ 边界 | 28nm @ 200MHz 典型动态功耗 5-15mW，预算 10mW 处于边界 |
| CD-005（低功耗 vs DFT） | 通过 | 无多电压域需求 |

**判定**：**PASS**

**exploration_template 应用详情**：

### 场景 4：需求矛盾场景

**输入描述**：28nm 工艺，目标频率 1GHz，面积上限 8kGates，功耗预算 0.5mW，CRC 校验模块

**预期路径**：`complete` → stage0 → stageA → stageB → stageC0（矛盾检测）→ 调和方案

**执行过程**：

| 步骤 | 执行内容 | 结果 |
|------|----------|------|
| 代办清单 | 输出待办事项清单 | ✅ |
| stage0 | 探略执行 | ✅ |
| stageA | 4 问确认 | ✅ |
| stageB | 9 项逐项确认 | ✅ |
| stageC0 | 矛盾检测 | ✅ 4 项矛盾检测到 |
| 调和方案 | 输出多方案供选择 | ✅ |

**矛盾检测详情**：

| 矛盾编号 | 涉及 REQ | 矛盾描述 | reference_values 引用 |
|----------|----------|----------|----------------------|
| CD-001 | REQ-001 | 28nm 要求 1GHz，超出典型 Fmax 400-500MHz | ✅ 正确引用 28nm 极限约 600MHz |
| CD-002 | REQ-001 + REQ-004 | CRC 多步运算需要多周期，1GHz 下 ≤1cycle 不可行 | ✅ 正确引用 multi_step_compute 规则 |
| CD-003 | REQ-002 + REQ-005 | AXI-Lite 接口 3-5kGates + CRC 功能超出 8kGates | ✅ 正确引用 axi_lite_slave 3-5kGates |
| CD-004 | REQ-001 + REQ-005 | 28nm @ 1GHz 典型功耗远超 0.5mW | ✅ 正确引用 28nm_500mhz_typical 15-40mW |

**调和方案输出**：

| 方案 | 调整方向 | 影响 |
|------|----------|------|
| A | 降频至 400MHz | 满足 CD-001，延迟增大 |
| B | 换 7nm 工艺 | 满足频率，成本增加 |
| C | 放宽面积至 20kGates | 满足 CD-003 |
| D | 放宽功耗至 15mW | 满足 CD-004 |

**判定**：**PASS**

| 版本 | 变更项 | 原值 | 新值 | 变更原因 |

## 4. 关键发现

### 4.1 优势

1. **流程完整性**：6 个场景覆盖 complete/partial/vague 三条路径，stage0→A→B→C0→C 全链路正确执行
2. **JSON+Skill 架构有效**：84 行核心 + 按需 Read JSON，模型理解准确，流程执行无偏差
3. **矛盾检测精准**：CD-001~CD-005 全部正确触发，reference_values 准确引用工艺 Fmax 和典型功耗
4. **确认规则可靠**：4 型分类（confirm/reject_turn/reject_hesitant/reject_partial）正确应用
5. **变更处理完整**：日志→重走→重汇总→重确认→重方案五步流程严格执行

### 4.2 待改进项

1. **CD-004 边界检测精度**：场景 1 中 28nm@200MHz<10mW 处于边界但未明确标注风险等级
2. **exploration_template 可选项数量**：部分场景仅给 2 个可选项，未达到模板要求的 3 个
