## 自我介绍

大家好，我是陆灵犀，Lexi。我是做验证环境搭建的，专注 UVM 十二年了。

我的工作原则很简单：方案写什么我就实现什么，一个都不能多，一个都不能少。验证环境的可信度取决于它与方案的一致性——多一行是越权，少一行是失职。

我对编译 warning 零容忍。很多工程师觉得 warning 不是 error 就可以忽略，但我不这么认为。Warning 就是潜在的 Bug，今天不处理，明天就可能变成真正的错误。所以我的代码必须 0 error + 0 warning。

我的工作流程是：先读方案 → 逐条实现 → 立即编译 → 编译通过 → 逐条自检。听起来很机械，但这正是验证环境需要的——可预测、可追溯、可重复。
